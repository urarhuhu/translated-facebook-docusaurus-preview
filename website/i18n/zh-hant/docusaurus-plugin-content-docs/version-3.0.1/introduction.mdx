# 介紹

⚡️ Docusaurus 能協助您快速打造**精美的文件網站**。

💸 自建技術堆疊成本高昂。請專注於**內容創作**，只需撰寫 Markdown 檔案即可。

💥 需要更多功能？可使用**進階特性**如版本控制、國際化、搜尋與主題客製化。

💅 查看**[精選 Docusaurus 網站](/showcase?tags=favorite)**獲取靈感，並閱讀**[用戶見證](https://x.com/sebastienlorber/timelines/1392048416872706049)**。

🧐 Docusaurus 是**靜態網站生成器**。它建構具備快速客戶端導航的**單頁應用程式**，充分發揮 **React** 的互動性優勢。除內建**文件功能**外，亦可用於建立**任何類型網站**（個人網站、產品頁、部落格、行銷登陸頁等）。

![](/img/slash-introducing.svg)

## 快速入門 ⏱️ {#fast-track}

透過實作，**5分鐘**快速理解 Docusaurus！

新建 Docusaurus 網站並遵循簡短的內建教學。

安裝 [Node.js](https://nodejs.org/en/download/) 後建立新網站：

```bash
npx create-docusaurus@latest my-website classic
```

啟動網站：

```bash
cd my-website
npx docusaurus start
```

開啟 [`http://localhost:3000`](http://localhost:3000) 並跟隨教學指引。

:::tip

立即透過 **[docusaurus.new](https://docusaurus.new)** 在瀏覽器中測試 Docusaurus！

或線上閱讀 **[5分鐘教學](https://tutorial.docusaurus.io)**。

:::

## 從 v1 遷移 {#migrating-from-v1}

Docusaurus v2+ 是基於現代化工具鏈的全面重寫版本。自 [v2 正式發布](https://docusaurus.io/blog/2022/08/01/announcing-docusaurus-2.0) 後，我們強烈建議**使用 v2+ 而非已停維護的 v1 版**。

眾多用戶已採用 v2+（參見 [案例展示](/showcase) 與 [趨勢分析](https://www.npmtrends.com/docusaurus-vs-@docusaurus/core)）。

**適用 v2+ 的情境：**

- :white_check_mark: 需建置現代 Jamstack 文件網站
- :white_check_mark: 需具備客戶端路由的單頁應用 (SPA)
- :white_check_mark: 需完整 React 與 MDX 功能
- :white_check_mark: 無需支援 IE11

**適用 [v1 版](https://v1.docusaurus.io/) 的情境：**

- :x: 不需單頁應用 (SPA)
- :x: 需支援 IE11（注意：IE [已終止支援](https://docs.microsoft.com/en-us/lifecycle/products/internet-explorer-11)）

現有 v1 用戶可參閱 [遷移指南](./migration/index.mdx) 升級至 v2+。

## 功能特色 {#features}

Docusaurus 的設計高度重視開發者與貢獻者的使用體驗。

- ⚛️ **以 React 與 💚 打造**：
  - 可透過 React 進行擴展與客製化
  - 透過提供自訂 React 元件，完全掌控網站的瀏覽體驗
- **模組化設計**：
  - 使用基礎模板快速建立網站，再透過進階功能與插件強化
  - 可將插件開源供社群共享
- ✂️ **開發者體驗**：
  - 立即開始撰寫文件
  - 通用配置入口點，便於貢獻者維護
  - 變更時支援熱重載與閃電般快速的增量建置
  - 基於路由的程式碼與資料分割
  - 輕鬆部署至 GitHub Pages、Netlify、Vercel 等服務

我們的共同目標是幫助使用者快速找到所需內容並更好地理解您的產品。我們分享最佳實踐，助您打造優質文件網站。

- 🎯 **SEO 友善**：
  - 為所有可能路徑靜態生成 HTML 檔案
  - 頁面專屬 SEO 優化，讓使用者直接找到與問題相關的官方文件
- 📝 **基於 MDX**：
  - 透過嵌入 Markdown 的 JSX 與 React 撰寫互動元件
  - 透過即時編輯器分享程式碼，讓使用者立即愛上您的產品
- 🔍 **搜尋功能**：全站內容皆可搜尋
- 💾 **文件版本控制**：保持文件與專案版本同步
- 🌍 **國際化 (i18n)**：支援多語言站點翻譯

Docusaurus v2+ 天生就具備極致速度，並貼心地為所有使用者提供無障礙訪問。

- ⚡️ **閃電般快速**。Docusaurus v2+ 遵循 [PRPL 模式](https://developers.google.com/web/fundamentals/performance/prpl-pattern/)，確保內容載入速度極快。
- 🦖 **無障礙設計**。注重可訪問性，讓所有使用者都能平等使用您的網站。

## 設計原則 {#design-principles}

- **低學習門檻**。Docusaurus 應易於學習和使用，因其 API 相當精簡。即使需要更多程式碼和時間，使用者仍能實現大多數功能。與其提供錯誤的抽象，不如不提供抽象，我們不希望使用者需要繞過錯誤的抽象來解決問題。必看演講—[最小 API 表面積](https://www.youtube.com/watch?v=4anAwXYqLG8)。
- **直觀性**。使用者在查看 Docusaurus 專案目錄或新增功能時不會感到不知所措。它應該看起來直觀且易於擴展，使用他們熟悉的方法。
- **分層架構**。堆疊各層（內容/主題/樣式）之間的關注點分離應清晰明確——抽象良好且模組化。
- **合理預設值**。常見且受歡迎的效能優化和配置將為使用者完成，但他們可以選擇覆蓋這些設定。
- **無供應商鎖定**。不強制使用者使用預設插件或 CSS，但強烈建議使用。某些核心基礎設施（如 React Loadable 和 React Router）不可替換，因為我們對其進行了預設效能優化，但更高層級的則可以。Markdown 引擎、CSS 框架、CSS 方法論和其他架構的選擇完全由使用者決定。

我們相信，作為開發者，了解函式庫的運作方式能幫助我們更好地使用它。因此，我們致力於解釋 Docusaurus 的架構和各個元件，希望使用者閱讀後能更深入地理解該工具並更熟練地使用它。

## 與其他工具的比較 {#comparison-with-other-tools}

在所有靜態網站生成器中，Docusaurus 獨特地專注於文件網站，並提供許多開箱即用的功能。

我們也研究了其他主要靜態網站生成器，並希望分享我們的比較見解，幫助您在眾多選擇中找到適合的工具。

### Gatsby {#gatsby}

[Gatsby](https://www.gatsbyjs.com/) 具備豐富功能，擁有龐大的插件生態系統，能夠實現 Docusaurus 的所有功能。當然，這也意味著更高的學習曲線。Gatsby 在多方面表現出色，適合構建各種類型的網站。而 Docusaurus 則專注於將一件事做到極致——成為編寫和發布內容的最佳工具。

GraphQL 是 Gatsby 的核心技術之一，儘管構建 Gatsby 網站不一定需要使用 GraphQL。在大多數靜態網站開發場景中，您並不需要 GraphQL 提供的靈活性。

Docusaurus v2+ 的許多設計靈感來自 Gatsby 的優秀特性，它是一個極佳的替代方案。

[Docz](https://github.com/pedronauck/docz) 是基於 Gatsby 的主題，用於構建文檔網站。目前其功能尚不如 Docusaurus 完善。

### Next.js {#nextjs}

[Next.js](https://nextjs.org/) 是另一個非常流行的混合式 React 框架。雖然它能幫助構建優秀的文檔網站，但並未針對文檔使用場景進行專門設計，需要投入更多工作量才能實現 Docusaurus 開箱即用的功能。

[Nextra](https://github.com/shuding/nextra) 是基於 Next.js 的約定式靜態網站生成器。目前其功能尚不如 Docusaurus 完善。

### VuePress {#vuepress}

[VuePress](https://vuepress.vuejs.org/) 與 Docusaurus 有許多相似之處——兩者都專注於內容為中心的網站，並提供開箱即用的文檔功能。但 VuePress 基於 Vue 構建，而 Docusaurus 基於 React。如果您需要基於 Vue 的解決方案，VuePress 會是不錯的選擇。

### MkDocs {#mkdocs}

[MkDocs](https://www.mkdocs.org/) 是流行的 Python 靜態網站生成器，其價值主張與 Docusaurus 相似。

如果您不需要單頁應用程序且不計劃使用 React，這是一個不錯的選擇。

[Material for MkDocs](https://squidfunk.github.io/mkdocs-material/) 是一個美觀的主題。

### Docsify {#docsify}

[Docsify](https://docsify.js.org/) 便於創建文檔網站，但它不是靜態網站生成器，也不支持 SEO 優化。

### GitBook {#gitbook}

[GitBook](https://www.gitbook.com/) 擁有簡潔的設計，曾被許多開源項目使用。隨著其重心轉向商業產品而非開源工具，許多功能已不再符合開源項目文檔站點的需求。因此許多用戶轉向了其他產品。您可以閱讀 Redux 遷移至 Docusaurus 的[相關討論](https://github.com/reduxjs/redux/issues/3161)。

目前 GitBook 僅對開源和非營利團隊免費。Docusaurus 則對所有人免費。

### Jekyll {#jekyll}

[Jekyll](https://github.com/jekyll/jekyll) 是最成熟的靜態網站生成器之一——事實上在 Docusaurus 出現之前，Facebook 多數開源網站都基於 Jekyll 構建！它的入門極其簡單。我們希望提供與使用 Jekyll 構建靜態網站相似的開發體驗。

相較於使用靜態生成 HTML 並通過 `<script />` 標籤添加交互性的方式，Docusaurus 網站是 React 應用程序。通過現代 JavaScript 生態系統工具，我們希望為文檔站點的性能、資源構建流程與優化，以及易用性設立新標準。
