# 📦 create-docusaurus

一個腳手架工具，可幫助您快速建立功能完整的 Docusaurus 應用程式。

## 使用方法 {#usage}

```bash
npx create-docusaurus@latest [name] [template] [rootDir]
```

`name` 參數將同時作為網站路徑及所建立應用程式中 package.json 的 `name` 欄位。可以是絕對路徑或相對於 `rootDir` 的路徑。

`template` 參數可為以下選項之一：

- `classic`: 使用經典模板（推薦）
- `facebook`: 使用 Facebook/Meta 模板，包含 Meta 特定的設定
- Git 倉庫網址（以 `https://` 或 `git@` 開頭），可克隆至目標位置
- 相對於當前工作目錄的本地檔案路徑，其內容將被複製至目標位置

`rootDir` 將用於解析網站目錄的絕對路徑，預設為當前工作目錄。

:::warning

建議在互動式 shell 中使用此命令以確保所有功能可用。

:::

## 選項 {#options}

### `-t, --typescript` {#typescript}

當模板參數為可識別名稱時使用。目前僅 `classic` 提供 TypeScript 變體。

### `-g, --git-strategy` {#git-strategy}

當模板參數為 Git 倉庫時使用。需為以下選項之一：

- `deep`: 保留完整 Git 歷史記錄
- `shallow`: 使用 `--depth=1` 進行淺層克隆
- `copy`: 進行淺層克隆但不建立 Git 倉庫
- `custom`: 輸入自訂 Git 克隆指令。我們將提示您輸入，例如可輸入 `git clone --depth 10`，我們會自動附加倉庫網址與目標目錄。

### `-p, --package-manager` {#package-manager}

值應為 `npm`、`yarn`、`pnpm` 或 `bun` 之一。若未明確提供，Docusaurus 將根據以下條件推斷：

- 當前工作目錄中已存在的鎖定檔案（例如在現有專案中建立網站時）
- 執行 `create-docusaurus` 時使用的指令（例如 `npm init`、`npx`、`yarn create`、`bunx` 等）
- 當所有啟發式條件都不存在時，透過互動式提示選擇

### `-s, --skip-install` {#skip-install}

若提供此選項，Docusaurus 在建立應用程式後不會自動安裝相依套件。`--package-manager` 選項僅在實際安裝相依套件時有用。
