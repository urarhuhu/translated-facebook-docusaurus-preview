# 📦 plugin-debug

```mdx-code-block
import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';
```

除錯插件會在 [`http://localhost:3000/__docusaurus/debug`](http://localhost:3000/__docusaurus/debug) 顯示實用的除錯資訊。

該插件主要對插件開發者有用，能更輕鬆地檢查 `.docusaurus` 資料夾內容（例如生成的路由），同時也能檢查不會寫入磁碟的資料結構，例如透過 `contentLoaded` 生命週期載入的插件資料。

:::info

若透過經典預設使用此插件，預設情況下該預設會**在開發環境啟用插件，在生產環境停用** (`debug: undefined`)，以避免暴露潛在敏感資訊。您可使用 `debug: true` 強制啟用，或 `debug: false` 強制停用。

若使用獨立插件，可能需要透過檢查環境變數達到相同效果：

```js title="docusaurus.config.js"
export default {
  plugins: [
    // highlight-next-line
    process.env.NODE_ENV === 'production' && '@docusaurus/plugin-debug',
  ].filter(Boolean),
};
```

:::

:::note

若您回報錯誤，我們可能會要求您在生產環境啟用此插件，以便更輕鬆檢查您的部署配置。

若無任何敏感資訊，您可保持生產環境啟用狀態 [如同我們的做法](/__docusaurus/debug)。

:::

## 安裝 {#installation}

```bash npm2yarn
npm install --save @docusaurus/plugin-debug
```

:::tip

若使用 `@docusaurus/preset-classic` 預設，則無需將此插件安裝為依賴項。

您可透過 [預設選項](../../using-plugins.mdx#docusauruspreset-classic) 配置此插件。

:::

## 配置 {#configuration}

此插件目前沒有可配置選項。

### 配置範例 {#ex-config}

您可透過預設選項或插件選項配置此插件。

:::tip

多數 Docusaurus 使用者透過預設選項配置此插件。

:::

```mdx-code-block
<Tabs groupId="api-config-ex">
<TabItem value="preset" label="Preset options">
```

若使用預設配置，請透過 [預設選項](../../using-plugins.mdx#docusauruspreset-classic) 配置此插件：

```js title="docusaurus.config.js"
export default {
  presets: [
    [
      '@docusaurus/preset-classic',
      {
        // highlight-next-line
        debug: true, // This will enable the plugin in production
      },
    ],
  ],
};
```

```mdx-code-block
</TabItem>
<TabItem value="plugin" label="Plugin Options">
```

若使用獨立插件，請直接提供選項給插件：

```js title="docusaurus.config.js"
export default {
  // highlight-next-line
  plugins: ['@docusaurus/plugin-debug'],
};
```

</TabItem>
