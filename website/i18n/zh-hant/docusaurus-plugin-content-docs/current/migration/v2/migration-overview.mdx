# 概述

本文檔將引導您將現有的 Docusaurus 1 網站遷移至 Docusaurus 2。

我們盡力簡化此過程，並提供遷移 CLI 工具。

## 主要差異 {#main-differences}

Docusaurus 1 是純粹的文檔網站生成器，使用 React 作為伺服器端模板引擎，但不在瀏覽器中載入 React。

Docusaurus 2 從頭重建，生成單頁應用程式，在瀏覽器中充分發揮 React 的威力。它提供了更高的可定制性，同時保留了 Docusaurus 1 的精華——易於上手、版本化文檔和國際化支援。

除此之外，Docusaurus 2 是一個**高效能的靜態網站生成器**，可快速建立常見的內容驅動型網站（例如文檔、部落格、產品落地頁和行銷頁面等）。

雖然我們的主要重點仍是協助您完善文檔，但由於 Docusaurus 2 本質上是 React 應用程式，**現在可用於構建任何類型的網站，不僅限於文檔網站**。

## Docusaurus 1 結構 {#docusaurus-1-structure}

您的 Docusaurus 1 網站應具有以下結構：

```bash
├── docs
└── website
    ├── blog
    ├── core
    │   └── Footer.js
    ├── package.json
    ├── pages
    ├── sidebars.json
    ├── siteConfig.js
    └── static
```

## Docusaurus 2 結構 {#docusaurus-2-structure}

遷移後，您的 Docusaurus 2 網站可能如下：

```sh
├── docs
└── website
    ├── blog
    ├── src
    │   ├── components
    │   ├── css
    │   └── pages
    ├── static
    ├── package.json
    ├── sidebars.json
    ├── docusaurus.config.js
```

:::info

此遷移不會變更 `/docs` 資料夾位置，但 Docusaurus v2 網站通常將 `/docs` 資料夾置於 `/website` 內

遷移至 v2 後，您可以自由將 `/docs` 資料夾放置於任何位置。

:::

## 遷移流程 {#migration-process}

要獲得功能完整的 Docusaurus 2 網站，需遷移多項內容：

- 套件
- CLI 指令
- 網站配置
- Markdown 檔案
- 側邊欄檔案
- 頁面、元件與 CSS
- 版本化文檔
- 國際化支援 🚧

## 自動化遷移流程 {#automated-migration-process}

[遷移 CLI](./migration-automated.mdx) 將為您處理多項遷移工作。

但部分內容難以自動化，您仍需手動完成。

:::note

建議先執行遷移 CLI，再透過手動遷移流程補齊缺失部分。

:::

## 手動遷移流程 {#manual-migration-process}

部分遷移內容（特別是頁面）無法自動化，需手動遷移。

[手動遷移指南](./migration-manual.mdx) 將提供所有手動步驟。
