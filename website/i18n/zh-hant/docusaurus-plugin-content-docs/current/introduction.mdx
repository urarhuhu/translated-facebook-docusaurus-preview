# 簡介

⚡️ Docusaurus 能協助您快速打造**精美的文件網站**。

💸 自建技術堆疊成本高昂。請專注於**內容創作**，只需撰寫 Markdown 檔案即可。

💥 需要更多功能？可使用**進階特性**如版本控制、國際化、搜尋及主題客製化。

💅 查看**[精選 Docusaurus 網站](/showcase?tags=favorite)**獲取靈感。

🧐 Docusaurus 是**靜態網站生成器**。它建構具備快速客戶端導覽的**單頁應用程式**，充分發揮 **React** 的互動能力。除預設**文件功能**外，亦適用於建立**任何類型網站**（個人網站、產品頁、部落格、行銷登陸頁等）。

![](/img/slash-introducing.svg)

## 快速入門 ⏱️ {#fast-track}

透過實作，**5分鐘**快速理解 Docusaurus！

新建 Docusaurus 網站並遵循**極簡**嵌入式教學。

安裝 [Node.js](https://nodejs.org/en/download/) 後建立新 Docusaurus 網站：

```bash
npx create-docusaurus@latest my-website classic
```

啟動網站：

```bash
cd my-website
npx docusaurus start
```

開啟 [`http://localhost:3000`](http://localhost:3000) 並跟隨教學。

:::tip

立即透過 **[docusaurus.new](https://docusaurus.new)** 在瀏覽器中測試 Docusaurus！

或線上閱讀 **[5分鐘教學](https://tutorial.docusaurus.io)**。

:::

## 從 v1 遷移 {#migrating-from-v1}

Docusaurus v2+ 是基於現代化工具鏈的完全重寫版本。自 [v2 正式發布](https://docusaurus.io/blog/2022/08/01/announcing-docusaurus-2.0) 後，我們強烈建議**使用 Docusaurus v2+ 而非已棄用的 v1 版**。

已有 [眾多用戶](/showcase) 採用 Docusaurus v2+（[趨勢分析](https://www.npmtrends.com/docusaurus-vs-@docusaurus/core)）。

**適用 Docusaurus v2+ 的情境：**

- :white_check_mark: 需建置現代化 Jamstack 文件網站
- :white_check_mark: 需具備客戶端路由的單頁應用程式 (SPA)
- :white_check_mark: 需完整 React 與 MDX 功能
- :white_check_mark: 無需支援 IE11

**適用 [Docusaurus v1](https://v1.docusaurus.io/) 的情境：**

- :x: 不需單頁應用程式 (SPA)
- :x: 需支援 IE11（注意：IE [已終止支援](https://docs.microsoft.com/en-us/lifecycle/products/internet-explorer-11)）

現有 v1 用戶可參閱我們的 [遷移指南](./migration/index.mdx) 升級至 v2+。

## 功能特性 {#features}

Docusaurus 的設計高度重視開發者與貢獻者的使用體驗。

- ⚛️ **以 💚 與 React 打造**：
  - 透過 React 擴展與自訂功能
  - 提供自訂 React 元件，完整掌控網站的瀏覽體驗
- 🔌 **可外掛擴充**：
  - 透過基礎模板快速建立網站，再使用進階功能與外掛
  - 將外掛開源以與社群共享
- ✂️ **開發者體驗**：
  - 立即開始撰寫文件
  - 通用設定入口點，讓貢獻者更易維護
  - 變更時支援熱重載與閃電般快速的增量建置
  - 基於路由的程式碼與資料分割
  - 輕鬆部署至 GitHub Pages、Netlify、Vercel 等服務

我們的共同目標——協助使用者快速找到所需內容並更深入理解您的產品。我們分享最佳實踐，助您打造優質文件網站。

- 🎯 **SEO 友善**：
  - 為所有可能路徑靜態生成 HTML 檔案
  - 頁面專屬 SEO 優化，讓使用者直接找到相關文件解決問題
- 📝 **基於 MDX**：
  - 透過嵌入 Markdown 的 JSX 與 React 撰寫互動元件
  - 透過即時編輯器分享程式碼，讓使用者當場愛上您的產品
- 🔍 **搜尋功能**：全站內容皆可搜尋
- 💾 **文件版本控制**：保持文件與專案版本同步
- 🌍 **國際化 (i18n)**：支援多語言網站翻譯

Docusaurus v2+ 天生就為所有使用者提供貼心的無障礙體驗與閃電般的速度。

- ⚡️ **極速效能**。Docusaurus v2+ 遵循 [PRPL 模式](https://developers.google.com/web/fundamentals/performance/prpl-pattern/)，確保內容載入快如閃電。
- 🦖 **無障礙設計**。注重可訪問性，讓所有使用者都能平等使用您的網站。

## 設計原則 {#design-principles}

- **學習曲線低**。Docusaurus API 簡潔易學易用。即使需要更多程式碼與時間，大多數功能仍可實現。與其提供錯誤的抽象層，不如不提供抽象層，我們不希望使用者需要繞過錯誤的抽象層來解決問題。必看演講—[最小 API 表面積](https://www.youtube.com/watch?v=4anAwXYqLG8)。
- **直覺化**。查看 Docusaurus 專案目錄或新增功能時，使用者不會感到不知所措。它應該看起來直覺且易於擴展，使用熟悉的開發方式。
- **分層架構**。堆疊各層（內容/主題/樣式）之間的關注點分離應清晰——抽象良好且模組化。
- **合理預設值**。為使用者預設執行常見的效能優化與設定，但仍可自行覆寫。
- **無廠商鎖定**。不強制使用預設外掛或 CSS（但強烈建議使用）。核心基礎設施如 React Loadable 和 React Router 因預設效能優化而不可替換，但更高層次的組件可以。Markdown 引擎、CSS 框架、CSS 方法論等架構選擇完全由使用者決定。

我們相信，作為開發者，了解函式庫的運作原理能幫助我們更熟練地使用它。因此我們投入心力解釋 Docusaurus 的架構與各項組件，希望使用者閱讀後能更深入理解工具，並更精通其使用方式。

## 與其他工具的比較 {#comparison-with-other-tools}

在所有靜態網站生成器中，Docusaurus 獨特聚焦於文件網站，並提供眾多開箱即用的功能。

我們也研究其他主流靜態網站生成器，希望透過比較分析，協助您在眾多選擇中找到適合的工具。

### Gatsby {#gatsby}

[Gatsby](https://www.gatsbyjs.com/) 具備豐富的功能，擁有龐大的插件生態系統，能夠實現 Docusaurus 的所有功能。當然，這也意味著更高的學習曲線。Gatsby 在許多方面表現出色，適合構建各種類型的網站。而 Docusaurus 則專注於一件事——成為撰寫和發布內容的最佳工具。

GraphQL 是 Gatsby 的核心部分，儘管構建 Gatsby 網站不一定需要使用 GraphQL。在大多數靜態網站開發場景中，您並不需要 GraphQL 提供的靈活性。

Docusaurus v2+ 的許多設計靈感來自 Gatsby 的優點，它是一個出色的替代方案。

[Docz](https://github.com/pedronauck/docz) 是一個基於 Gatsby 的主題，用於構建文檔網站。目前它的功能比 Docusaurus 少。

### Next.js {#nextjs}

[Next.js](https://nextjs.org/) 是另一個非常流行的混合式 React 框架。它可以幫助您構建一個優秀的文檔網站，但它並非專為文檔場景設計，因此需要更多工作才能實現 Docusaurus 開箱即用的功能。

[Nextra](https://github.com/shuding/nextra) 是一個基於 Next.js 的靜態網站生成器，具有明確的設計理念。目前它的功能比 Docusaurus 少。

### VitePress {#vitepress}

[VitePress](https://vitepress.dev/) 與 Docusaurus 有許多相似之處——兩者都專注於內容為主的網站，並提供開箱即用的文檔功能。然而，VitePress 基於 Vue，而 Docusaurus 基於 React。如果您需要基於 Vue 的解決方案，VitePress 是一個不錯的選擇。

### MkDocs {#mkdocs}

[MkDocs](https://www.mkdocs.org/) 是一個流行的 Python 靜態網站生成器，其價值主張與 Docusaurus 相似。

如果您不需要單頁應用程序且不打算使用 React，這是一個不錯的選擇。

[Material for MkDocs](https://squidfunk.github.io/mkdocs-material/) 是一個美觀的主題。

### Docsify {#docsify}

[Docsify](https://docsify.js.org/) 可以輕鬆創建文檔網站，但它不是靜態網站生成器，也不利於 SEO。

### GitBook {#gitbook}

[GitBook](https://www.gitbook.com/) 設計簡潔，曾被許多開源項目使用。隨著其重心轉向商業產品而非開源工具，許多功能已不再符合開源項目文檔網站的需求。因此，許多項目轉向了其他工具。您可以閱讀 Redux 遷移至 Docusaurus 的[討論](https://github.com/reduxjs/redux/issues/3161)。

目前，GitBook 僅對開源和非營利團隊免費。Docusaurus 對所有人免費。

### Jekyll {#jekyll}

[Jekyll](https://github.com/jekyll/jekyll) 是最成熟的靜態網站生成器之一，曾是一個優秀的工具——事實上，在 Docusaurus 之前，Facebook 的大多數開源網站都是基於 Jekyll 構建的！它的入門非常簡單。我們希望為用戶提供類似於使用 Jekyll 構建靜態網站的開發體驗。

與靜態生成 HTML 並通過 `<script />` 標籤添加交互性的方式相比，Docusaurus 網站是 React 應用程序。我們希望通過現代 JavaScript 生態系統工具，為文檔網站的性能、資源構建流程和優化，以及設置簡便性樹立新標準。

### Rspress {#rspress}

[Rspress](https://rspress.dev/) 是一個基於 Rspack（Rust 驅動的打包工具）的快速靜態網站生成器。它支援使用 MDX（內嵌 React 組件的 Markdown）編寫內容，整合全文搜索、多語言支援（i18n），並可透過插件擴展功能。專為打造優雅的文檔和靜態網站設計，Rspress 生成的靜態 HTML 檔案易於部署。

Rspress 與 Docusaurus 相當相似，各有優缺點。Rspress 創建時間較晚，受益於現代化基礎架構，能實現更快的網站構建速度。Docusaurus 則以成熟度、完整功能集、靈活性及強大社群見長，並持續[現代化其基礎架構](https://github.com/facebook/docusaurus/issues/10556)以保持性能競爭力。
