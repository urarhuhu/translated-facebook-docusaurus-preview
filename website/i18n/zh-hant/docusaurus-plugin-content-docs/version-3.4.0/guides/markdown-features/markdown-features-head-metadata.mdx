# 頁首元數據

## 自訂頁首元數據 {#customizing-head-metadata}

Docusaurus 會自動為您設置 `<html>`、`<head>` 和 `<body>` 中有用的頁面元數據。您可以在 Markdown 文件中使用 `<head>` 標籤來添加額外的元數據（或覆蓋現有的元數據）：

```md title="markdown-features-head-metadata.mdx"
---
id: head-metadata
title: Head Metadata
---

<!-- highlight-start -->
<head>
  <html className="some-extra-html-class" />
  <body className="other-extra-body-class" />
  <title>Head Metadata customized title!</title>
  <meta charSet="utf-8" />
  <meta name="twitter:card" content="summary" />
  <link rel="canonical" href="https://docusaurus.io/docs/markdown-features/head-metadata" />
</head>
<!-- highlight-end -->

# Head Metadata

My text
```

```mdx-code-block
<head>
  <html className="some-extra-html-class" />
  <body className="other-extra-body-class" />
  <title>Head Metadata customized title!</title>
  <meta charSet="utf-8" />
  <meta name="twitter:card" content="summary" />
  <link rel="canonical" href="https://docusaurus.io/docs/markdown-features/head-metadata" />
</head>
```

此 `<head>` 聲明已作為示範添加到當前 Markdown 文檔中。打開瀏覽器的開發者工具，查看此頁面的元數據如何受到影響。

:::note

此功能基於 Docusaurus 的 [`<Head>`](./../../docusaurus-core.mdx#head) 組件構建。詳盡文檔請參考 [react-helmet](https://github.com/nfl/react-helmet)。

:::

:::tip[常規 SEO 無需手動設置]

內容插件（例如文檔和博客）提供 `description`、`keywords` 和 `image` 等前置元數據選項，這些選項會自動應用於 `description` 和 `og:description`，而使用 `<head>` 標籤時您需要手動聲明兩個元數據標籤。

:::

## Markdown 頁面描述 {#markdown-page-description}

Markdown 頁面的描述元數據可能用於比頁首元數據更多的地方。例如，文檔插件的[自動生成分類索引頁](../docs/sidebar/items.mdx#generated-index-page)會使用描述元數據來顯示文檔卡片。

默認情況下，描述是第一行有實際內容的文字，並會嘗試將其轉換為純文本。例如，以下文件...

```md
# Title

Main content... May contain some [links](./file.mdx) or **emphasis**.
```

...將具有默認描述「主要內容...可能包含一些鏈接或強調」。然而，**這並非完全可靠的功能**。當它無法產生合理的描述時，您可以通過前置元數據明確提供一個：

```md
---
description: This description will override the default.
---

# Title

Main content... May contain some [links](./file.mdx) or **emphasis**.
```
