# 簡介

⚡️ Docusaurus 能協助您快速打造出**美觀的說明文件網站**。

💸 自建技術堆疊成本高昂。與其如此，不如**專注於內容**，只需撰寫 Markdown 檔案即可。

💥 準備好進階功能了嗎？使用**版本控制、國際化、搜尋和主題自訂**等進階功能。

💅 查看**[精選 Docusaurus 網站](/showcase?tags=favorite)**獲取靈感，並閱讀一些**[使用者見證](https://x.com/sebastienlorber/timelines/1392048416872706049)**。

🧐 Docusaurus 是**靜態網站生成器**。它建構了一個**單頁應用程式**，具備快速的客戶端導航功能，充分發揮**React**的優勢讓您的網站具有互動性。它提供開箱即用的**說明文件功能**，但也可用於建立**任何類型的網站**（個人網站、產品、部落格、行銷登陸頁面等）。

![](/img/slash-introducing.svg)

## 快速入門 ⏱️ {#fast-track}

透過實作在**5分鐘內**了解 Docusaurus！

建立一個新的 Docusaurus 網站並遵循**非常簡短**的內建教學。

安裝 [Node.js](https://nodejs.org/en/download/) 並建立新的 Docusaurus 網站：

```bash
npx create-docusaurus@latest my-website classic
```

啟動網站：

```bash
cd my-website
npx docusaurus start
```

開啟 [`http://localhost:3000`](http://localhost:3000) 並遵循教學。

:::tip

使用 **[docusaurus.new](https://docusaurus.new)** 立即在瀏覽器中測試 Docusaurus！

或線上閱讀 **[5分鐘教學](https://tutorial.docusaurus.io)**。

:::

## 從 v1 遷移 {#migrating-from-v1}

Docusaurus v2 是從 Docusaurus v1 完全重寫而來，採用了現代化的工具鏈。在 [v2 正式發布](https://docusaurus.io/blog/2022/08/01/announcing-docusaurus-2.0) 後，我們強烈建議您**使用 Docusaurus v2 而非 Docusaurus v1**，因為 Docusaurus v1 已被棄用。

已有 [許多使用者](/showcase) 正在使用 Docusaurus v2 ([趨勢](https://www.npmtrends.com/docusaurus-vs-@docusaurus/core))。

**以下情況請使用 Docusaurus v2：**

- :white_check_mark: 您需要一個現代的 Jamstack 說明文件網站
- :white_check_mark: 您需要一個具備客戶端路由的單頁應用程式 (SPA)
- :white_check_mark: 您需要完整的 React 和 MDX 功能
- :white_check_mark: 您不需要支援 IE11

**以下情況請使用 [Docusaurus v1](https://v1.docusaurus.io/)：**

- :x: 您不需要單頁應用程式 (SPA)
- :x: 您需要支援 IE11（...真的需要嗎？IE [已停止支援](https://docs.microsoft.com/en-us/lifecycle/products/internet-explorer-11)）

現有的 v1 使用者若想升級至 v2，可參閱我們的[遷移指南](./migration/migration-overview.mdx)。

## 功能特色 {#features}

Docusaurus 的設計高度重視開發者與貢獻者體驗。

- ⚛️ **以 React 與💚打造**：
  - 透過 React 進行擴展與客製化
  - 提供自訂 React 元件，完全掌控網站的瀏覽體驗
- **模組化設計**：
  - 使用基礎模板快速建站，再逐步加入進階功能與插件
  - 可開源插件與社群共享
- ✂️ **開發者體驗**：
  - 立即開始撰寫文件
  - 統一配置入口，便於貢獻者維護
  - 熱重載與閃電般快速的增量建置
  - 基於路由的程式碼與數據拆分
  - 輕鬆部署至 GitHub Pages、Netlify、Vercel 等服務

我們的共同目標——幫助使用者快速找到所需內容並理解您的產品。我們分享最佳實踐，助您打造優質文件網站。

- 🎯 **SEO 優化**：
  - 為所有可能路徑靜態生成 HTML 檔案
  - 頁面專屬 SEO 配置，讓使用者直接找到相關解決方案
- 📝 **基於 MDX**：
  - 在 Markdown 中嵌入 JSX 和 React 撰寫互動元件
  - 透過即時編輯器分享程式碼，讓使用者立即體驗產品魅力
- 🔍 **全文搜尋**：全站內容皆可搜尋
- 💾 **文件版本控制**：保持文件與專案版本同步
- 🌍 **國際化 (i18n)**：支援多語言站點翻譯

Docusaurus 2 天生具備極致效能與無障礙設計。

- ⚡️ **閃電速度**。遵循 [PRPL 模式](https://developers.google.com/web/fundamentals/performance/prpl-pattern/)，確保內容極速載入
- 🦖 **無障礙設計**。注重可訪問性，讓所有使用者平等獲取內容

## 設計原則 {#design-principles}

- **低學習曲線**：API 簡潔易學。即使需編寫更多程式碼，使用者仍能實現多數功能。寧缺毋濫，避免錯誤抽象導致繞道解決。必看演講—[最小化 API 表面積](https://www.youtube.com/watch?v=4anAwXYqLG8)
- **直覺化**：專案目錄結構清晰，新增功能符合直覺，採用熟悉的方式擴展
- **分層架構**：內容/主題/樣式各層關注點分離，模組化抽象設計
- **合理預設**：自動實施常見效能優化配置，同時保留覆寫選項
- **無廠商鎖定**：鼓勵使用預設插件與 CSS，但非強制要求。僅核心基礎設施（如 React Loadable 和 React Router）不可替換以維持效能優化，其他架構選擇（Markdown 引擎、CSS 框架等）完全開放

我們相信理解工具運作原理能提升使用效能，因此詳細說明 Docusaurus 架構與各組件，幫助使用者深入掌握。

## 與其他工具比較 {#comparison-with-other-tools}

在所有靜態網站生成器中，Docusaurus 專注於文件網站，提供眾多開箱即用功能。

我們研究過其他主流靜態網站生成器，願分享比較見解，助您在眾多選擇中找到合適工具。

### Gatsby {#gatsby}

[Gatsby](https://www.gatsbyjs.com/) 具備豐富功能，擁有龐大的插件生態系統，能實現 Docusaurus 的所有功能。但這自然伴隨著較高的學習曲線。Gatsby 在多方面表現出色，適合構建各類網站。而 Docusaurus 則專注於將一件事做到極致——成為編寫與發布內容的最佳工具。

GraphQL 是 Gatsby 的核心技術之一，但構建 Gatsby 網站時並非必須使用。在大多數靜態網站場景中，您並不需要 GraphQL 提供的靈活性。

Docusaurus 2 的許多設計靈感來自 Gatsby 的優秀特性，它是一個絕佳的替代方案。

[Docz](https://github.com/pedronauck/docz) 是基於 Gatsby 的主題，用於構建文檔網站。目前其功能尚不及 Docusaurus 完善。

### Next.js {#nextjs}

[Next.js](https://nextjs.org/) 是另一個熱門的混合式 React 框架。雖然可用於構建文檔網站，但並未針對文檔場景進行專門設計，要實現 Docusaurus 的開箱即用功能需要更多開發工作。

[Nextra](https://github.com/shuding/nextra) 是基於 Next.js 的靜態網站生成器，目前功能完整度低於 Docusaurus。

### VuePress {#vuepress}

[VuePress](https://vuepress.vuejs.org/) 與 Docusaurus 有諸多相似之處——兩者都專注於內容型網站，並提供開箱即用的文檔功能。不同之處在於 VuePress 基於 Vue 構建，而 Docusaurus 基於 React。如需 Vue 技術棧方案，VuePress 是不錯的選擇。

### MkDocs {#mkdocs}

[MkDocs](https://www.mkdocs.org/) 是流行的 Python 靜態網站生成器，核心價值主張與 Docusaurus 相似。

如果您不需要單頁應用且不計劃使用 React，這是個理想的選擇。

[Material for MkDocs](https://squidfunk.github.io/mkdocs-material/) 提供了精美的主題方案。

### Docsify {#docsify}

[Docsify](https://docsify.js.org/) 能快速創建文檔網站，但並非靜態網站生成器且不利於 SEO 優化。

### GitBook {#gitbook}

[GitBook](https://www.gitbook.com/) 擁有簡潔的設計，曾被眾多開源項目採用。隨著其轉型為商業產品，許多功能已不再符合開源文檔站點的需求，促使諸多項目遷移。可參閱 [Redux 遷移至 Docusaurus 的討論](https://github.com/reduxjs/redux/issues/3161)。

當前 GitBook 僅對開源和非營利團隊免費，而 Docusaurus 對所有用戶完全免費。

### Jekyll {#jekyll}

[Jekyll](https://github.com/jekyll/jekyll) 是最成熟的靜態網站生成器之一——事實上在 Docusaurus 之前，Facebook 多數開源網站都基於 Jekyll 構建！它的入門門檻極低。我們希望延續這種與 Jekyll 相似的開發體驗。

相較於通過 `<script />` 標籤添加交互性的靜態 HTML，Docusaurus 網站是完整的 React 應用。藉助現代 JavaScript 生態工具鏈，我們致力於為文檔站點設立性能、資源構建流程、優化方案和易用性方面的新標準。
