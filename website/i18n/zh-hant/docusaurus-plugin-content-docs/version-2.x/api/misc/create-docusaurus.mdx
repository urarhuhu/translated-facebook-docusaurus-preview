# 📦 create-docusaurus

一個用於快速建立功能完整 Docusaurus 應用程式的腳手架工具。

## 使用方法 {#usage}

```bash
npx create-docusaurus@latest [name] [template] [rootDir]
```

`name` 參數將同時作為網站路徑和所建立應用程式 package.json 中的 `name` 欄位。可以是絕對路徑或相對於 `rootDir` 的路徑。

`template` 參數可為以下選項之一：

- `classic`: 使用經典模板（推薦）
- `facebook`: 使用 Facebook/Meta 模板，包含 Meta 特定設定
- 以 `https://` 或 `git@` 開頭的 git 倉庫 URL，可克隆至目標位置
- 相對於當前工作目錄的本地文件路徑，其內容將被複製至目標位置

`rootDir` 將用於解析網站目錄的絕對路徑，預設為當前工作目錄。

:::warning

建議在互動式 shell 中使用此命令以確保所有功能可用。

:::

## 選項 {#options}

### `-t, --typescript` {#typescript}

當模板參數為可識別名稱時使用。目前僅 `classic` 模板提供 TypeScript 版本。

### `-g, --git-strategy` {#git-strategy}

當模板參數為 git 倉庫時使用，需為以下選項之一：

- `deep`: 保留完整 git 歷史記錄
- `shallow`: 使用 `--depth=1` 進行淺克隆
- `copy`: 執行淺克隆但不建立 git 倉庫
- `custom`: 輸入自訂 git 克隆指令。我們將提示您輸入，例如可輸入 `git clone --depth 10`，系統會自動附加倉庫 URL 和目標目錄。

### `-p, --package-manager` {#package-manager}

值應為 `npm`、`yarn` 或 `pnpm` 之一。若未明確指定，Docusaurus 將根據以下條件自動判斷：

- 當前工作目錄中已存在的鎖定文件（例如在現有專案中建立網站時）
- 執行 `create-docusaurus` 時使用的指令（如 `npm init`、`npx`、`yarn create` 等）
- 若無上述條件，則透過互動式提示選擇

### `-s, --skip-install` {#skip-install}

若指定此參數，Docusaurus 在建立應用程式後將不會自動安裝依賴項。`--package-manager` 選項僅在實際安裝依賴項時有效。
