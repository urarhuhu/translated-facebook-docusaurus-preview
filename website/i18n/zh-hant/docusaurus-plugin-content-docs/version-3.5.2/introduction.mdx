# 簡介

⚡️ Docusaurus 能協助您快速部署一個**美觀的技術文件網站**。

💸 自建技術堆疊成本高昂。請專注於**內容創作**，只需撰寫 Markdown 檔案即可。

💥 需要更多功能？可使用**進階特性**如版本控制、國際化、站內搜索與主題客製化。

💅 查看**[精選 Docusaurus 網站](/showcase?tags=favorite)**獲取靈感，或閱讀**[用戶見證](https://x.com/sebastienlorber/timelines/1392048416872706049)**。

🧐 Docusaurus 是**靜態網站生成器**。它構建具備快速客戶端導航的**單頁應用程式**，充分發揮 **React** 的互動性優勢。除預設的**技術文件功能**外，亦可用於建立**任何類型網站**（個人網站、產品頁、部落格、行銷登陸頁等）。

![](/img/slash-introducing.svg)

## 快速入門 ⏱️ {#fast-track}

透過實作在 **5 分鐘** 內掌握 Docusaurus！

建立新 Docusaurus 網站並跟隨簡短的內建教學。

安裝 [Node.js](https://nodejs.org/en/download/) 後建立新網站：

```bash
npx create-docusaurus@latest my-website classic
```

啟動網站：

```bash
cd my-website
npx docusaurus start
```

開啟 [`http://localhost:3000`](http://localhost:3000) 並跟隨教學指引。

:::tip

立即透過 **[docusaurus.new](https://docusaurus.new)** 在瀏覽器中測試 Docusaurus！

或線上閱讀 **[5 分鐘教學](https://tutorial.docusaurus.io)**。

:::

## 從 v1 遷移 {#migrating-from-v1}

Docusaurus v2+ 是基於現代化工具鏈的完全重構版本。自 [v2 正式發布](https://docusaurus.io/blog/2022/08/01/announcing-docusaurus-2.0) 後，我們強烈建議**使用 v2+ 而非已棄用的 v1 版本**。

眾多用戶已採用 v2+ 版本（參見 [案例展示](/showcase) 與 [趨勢分析](https://www.npmtrends.com/docusaurus-vs-@docusaurus/core)）。

**適用 Docusaurus v2+ 的情境：**

- :white_check_mark: 需要現代化 Jamstack 技術文件網站
- :white_check_mark: 需具備客戶端路由的單頁應用程式 (SPA)
- :white_check_mark: 需完整 React 與 MDX 功能支援
- :white_check_mark: 無需支援 IE11 瀏覽器

**適用 [Docusaurus v1](https://v1.docusaurus.io/) 的情境：**

- :x: 不需單頁應用程式架構
- :x: 需支援已終止維護的 IE11 瀏覽器（參見 [微軟生命週期公告](https://docs.microsoft.com/en-us/lifecycle/products/internet-explorer-11)）

現有 v1 用戶可參考 [遷移指南](./migration/index.mdx) 升級至 v2+ 版本。

## 功能特色 {#features}

Docusaurus 的設計高度重視開發者與貢獻者的使用體驗。

- ⚛️ **以 React 與 💚 打造**：
  - 透過 React 進行擴展與客製化
  - 透過自訂 React 元件全面掌控網站的瀏覽體驗
- 🔌 **可插拔式架構**：
  - 使用基礎模板快速建構網站，再透過進階功能與插件強化
  - 可將插件開源供社群共享使用
- ✂️ **開發者體驗**：
  - 立即開始撰寫文件
  - 統一配置入口提升協作者維護性
  - 熱重載配合閃電般快速的增量建構
  - 基於路由的程式碼與資料分割
  - 輕鬆部署至 GitHub Pages、Netlify、Vercel 等服務

我們的共同目標——幫助使用者快速找到所需內容並更好理解您的產品。我們分享最佳實踐，助您打造優質文件站點。

- 🎯 **SEO 友善**：
  - 為所有可能路徑靜態生成 HTML 檔案
  - 頁面專屬 SEO 優化，讓使用者直接找到與問題相關的官方文件
- 📝 **基於 MDX**：
  - 透過嵌入 Markdown 的 JSX 與 React 編寫互動元件
  - 透過即時編輯器分享程式碼，讓使用者當場愛上您的產品
- 🔍 **全站搜尋**：完整搜尋功能
- 💾 **文件版本控制**：保持文件與專案版本同步
- 🌍 **國際化 (i18n)**：支援多語言站點翻譯

Docusaurus v2+ 天生具備極致效能，並貼心顧及所有使用者的可訪問性。

- ⚡️ **閃電速度**。遵循 [PRPL 模式](https://developers.google.com/web/fundamentals/performance/prpl-pattern/)，確保內容極速載入
- 🦖 **無障礙設計**。注重可訪問性，讓所有使用者平等獲取內容

## 設計原則 {#design-principles}

- **低學習曲線**。因 API 精簡，Docusaurus 易學易用。即使需編寫更多程式碼，使用者仍能實現大多數功能。寧缺毋濫——與其提供錯誤抽象，不如不提供抽象，避免使用者被迫繞過設計缺陷。必看演講—[最小化 API 表面積](https://www.youtube.com/watch?v=4anAwXYqLG8)。
- **直覺化**。查看專案目錄或新增功能時，使用者不會感到無所適從。架構應符合直覺，易於基於熟悉的方式擴展。
- **分層架構**。各層級（內容/主題/樣式）關注點分離明確——抽象完善且模組化。
- **合理預設**。自動實施常見效能優化與配置，同時保留覆寫選項。
- **無廠商鎖定**。不強制使用預設插件或 CSS（但強烈建議）。核心基礎設施如 React Loadable 和 React Router 因預設效能優化不可替換，但更高層級組件可自由選擇。Markdown 引擎、CSS 框架、CSS 方法論等架構完全由使用者決定。

我們相信，作為開發者，理解工具運作原理能提升使用效率。因此我們投入精力解構 Docusaurus 的架構與各組件，幫助使用者深入掌握工具精髓。

## 與其他工具比較 {#comparison-with-other-tools}

在所有靜態網站生成器中，Docusaurus 獨特聚焦於文件站點，並提供豐富開箱即用功能。

我們亦研究其他主流靜態網站生成器，分享比較見解，協助您在眾多選擇中做出決策。

### Gatsby {#gatsby}

[Gatsby](https://www.gatsbyjs.com/) 具備豐富功能，擁有龐大的插件生態系統，能實現 Docusaurus 的所有功能。但這也意味著更高的學習曲線。Gatsby 擅長多種應用場景，適合構建各類網站；而 Docusaurus 則專注於一件事——成為編寫與發布內容的最佳工具。

GraphQL 是 Gatsby 的核心技術之一，但構建靜態網站時通常不需要其提供的靈活性。

Docusaurus v2+ 的許多設計靈感來自 Gatsby 的優點，它是一個出色的替代方案。

[Docz](https://github.com/pedronauck/docz) 是基於 Gatsby 的文檔網站主題，目前功能不如 Docusaurus 全面。

### Next.js {#nextjs}

[Next.js](https://nextjs.org/) 是另一款熱門的混合式 React 框架。雖然可用於構建文檔網站，但其設計並非專為文檔場景優化，實現 Docusaurus 開箱即用的功能需要更多開發工作。

[Nextra](https://github.com/shuding/nextra) 是基於 Next.js 的靜態網站生成器，目前功能不如 Docusaurus 完善。

### VitePress {#vitepress}

[VitePress](https://vitepress.dev/) 與 Docusaurus 高度相似——兩者都專注於內容型網站並提供開箱即用的文檔功能。不同之處在於 VitePress 基於 Vue，而 Docusaurus 基於 React。如需 Vue 技術棧方案，VitePress 是不錯的選擇。

### MkDocs {#mkdocs}

[MkDocs](https://www.mkdocs.org/) 是流行的 Python 靜態網站生成器，核心價值主張與 Docusaurus 相似。

若不需要單頁應用且不計劃使用 React，這是個理想的選擇。

[Material for MkDocs](https://squidfunk.github.io/mkdocs-material/) 提供了精美的主題方案。

### Docsify {#docsify}

[Docsify](https://docsify.js.org/) 便於快速創建文檔網站，但非靜態生成且對 SEO 不友好。

### GitBook {#gitbook}

[GitBook](https://www.gitbook.com/) 擁有簡潔設計，曾被眾多開源項目採用。隨著其轉向商業化產品定位，許多功能已不符合開源文檔站點需求，導致用戶遷移至其他方案（參閱 [Redux 遷移案例](https://github.com/reduxjs/redux/issues/3161)）。

當前 GitBook 僅對開源和非營利團隊免費，而 Docusaurus 完全免費開放。

### Jekyll {#jekyll}

[Jekyll](https://github.com/jekyll/jekyll) 是最成熟的靜態網站生成器之一——事實上在 Docusaurus 之前，Facebook 多數開源網站都基於 Jekyll 構建！其入門門檻極低，我們希望延續這種簡潔的開發體驗。

相較於通過 `<script />` 標籤添加交互性的靜態 HTML，Docusaurus 生成的是 React 應用。我們希望通過現代 JavaScript 工具鏈，在文檔站點性能、資源構建流程和優化易用性方面樹立新標準。
