# 自動化遷移

遷移 CLI 會自動將您的 v1 網站遷移至 v2 網站。

:::info

使用遷移 CLI 後仍需要手動操作，因為我們無法自動化完成整個遷移過程

:::

遷移 CLI 會遷移以下內容：

- 網站配置（從 `siteConfig.js` 遷移至 `docusaurus.config.js`）
- `package.json`
- `sidebars.json`
- `/docs`
- `/blog`
- `/static`
- 若您的網站使用版本控制功能，則會遷移 `versioned_sidebar.json` 和 `/versioned_docs`

使用遷移 CLI 的步驟如下：

1. 在使用遷移 CLI 前，請確認 `/docs`、`/blog`、`/static`、`sidebars.json`、`siteConfig.js`、`package.json` 符合預期結構。

2. 要遷移您的 v1 網站，請使用適當的檔案系統路徑執行遷移 CLI：

```bash
# migration command format
npx @docusaurus/migrate migrate <v1 website directory> <desired v2 website directory>

# example
npx @docusaurus/migrate migrate ./v1-website ./v2-website
```

3. 若要在本地預覽您的新網站，請進入 v2 網站的目錄並啟動開發伺服器。

```bash npm2yarn
cd ./v2-website
npm install
npm start
```

:::danger

遷移 CLI 會更新現有檔案。請務必先提交這些檔案的變更！

:::

#### 選項 {#options}

您可以為遷移 CLI 添加選項標記，以自動將 Markdown 內容和頁面遷移至 v2 版本。您可能仍需手動調整以達到理想結果。

| Name     | Description                                            |
| -------- | ------------------------------------------------------ |
| `--mdx`  | Add this flag to convert Markdown to MDX automatically |
| `--page` | Add this flag to migrate pages automatically           |

```bash
# example using options
npx @docusaurus/migrate migrate --mdx --page ./v1-website ./v2-website
```

:::danger

頁面與 MDX 的遷移功能仍在開發中。

建議您先在不使用這些選項的情況下執行遷移，提交變更後，再嘗試使用 `--page` 和 `--mdx` 選項重新執行遷移。

如此一來，您將能更輕鬆地檢查並修正差異。

:::
