# 簡介

⚡️ Docusaurus 能協助您快速打造出**精美的文件網站**。

💸 自建技術堆疊成本高昂。請專注於**內容創作**，只需撰寫 Markdown 檔案即可。

💥 想更進一步？可使用**進階功能**如版本控制、國際化(i18n)、搜尋與主題客製化。

💅 查看**[精選 Docusaurus 網站](/showcase?tags=favorite)**獲取靈感，或閱讀**[用戶見證](https://x.com/sebastienlorber/timelines/1392048416872706049)**。

🧐 Docusaurus 是**靜態網站生成器**。它建構出具有快速客戶端導航的**單頁應用程式(SPA)**，充分發揮 **React** 的互動性優勢。除內建**文件專屬功能**外，亦可用於建立**任何類型網站**（個人網站、產品頁、部落格、行銷登陸頁等）。

![](/img/slash-introducing.svg)

## 快速入門 ⏱️ {#fast-track}

透過實作，**5分鐘**快速理解 Docusaurus！

新建 Docusaurus 網站並遵循**極簡**嵌入式教學。

安裝 [Node.js](https://nodejs.org/en/download/) 後建立新站點：

```bash
npx create-docusaurus@latest my-website classic
```

啟動網站：

```bash
cd my-website
npx docusaurus start
```

開啟 [`http://localhost:3000`](http://localhost:3000) 跟隨教學操作。

:::tip

立即透過 **[docusaurus.new](https://docusaurus.new)** 在瀏覽器中測試 Docusaurus！

或線上閱讀 **[5分鐘教學](https://tutorial.docusaurus.io)**。

:::

## 從 v1 遷移 {#migrating-from-v1}

Docusaurus v2+ 是基於現代化工具鏈的完全重寫版本。自 [v2 正式發布](https://docusaurus.io/blog/2022/08/01/announcing-docusaurus-2.0) 後，我們強烈建議**使用 v2+ 而非已停維護的 v1 版本**。

眾多用戶已採用 v2+（參見 [案例展示](/showcase) 與 [趨勢分析](https://www.npmtrends.com/docusaurus-vs-@docusaurus/core)）。

**適用 v2+ 的情境：**

- :white_check_mark: 需建置現代化 Jamstack 文件網站
- :white_check_mark: 需具備客戶端路由的單頁應用(SPA)
- :white_check_mark: 需完整 React 與 MDX 功能支援
- :white_check_mark: 無需支援 IE11 瀏覽器

**適用 [v1](https://v1.docusaurus.io/) 的情境：**

- :x: 不需單頁應用架構
- :x: 需支援 IE11（注意：IE [已終止支援](https://docs.microsoft.com/en-us/lifecycle/products/internet-explorer-11)）

v1 用戶可參閱 [遷移指南](./migration/index.mdx) 升級至 v2+。

## 功能特色 {#features}

Docusaurus 的設計高度重視開發者與貢獻者的使用體驗。

- ⚛️ **以 React 與 💚 打造**：
  - 透過 React 進行擴展與客製化
  - 透過提供自訂 React 元件，完全掌控網站的瀏覽體驗
- **模組化設計**：
  - 使用基礎模板快速建立網站，再透過進階功能與插件強化
  - 可將插件開源供社群共享
- ✂️ **開發者體驗**：
  - 立即開始撰寫文件
  - 通用配置入口點，便於貢獻者維護
  - 變更時支援熱重載與閃電般快速的增量建置
  - 基於路由的程式碼與資料分割
  - 輕鬆部署至 GitHub Pages、Netlify、Vercel 等服務

我們的共同目標——協助使用者快速找到所需內容並更深入理解您的產品。我們分享最佳實踐，助您打造完善的文件網站。

- 🎯 **SEO 友善**：
  - 為所有可能路徑靜態生成 HTML 檔案
  - 頁面專屬 SEO 優化，讓使用者直接找到與其問題相關的官方文件
- 📝 **基於 MDX**：
  - 透過嵌入 Markdown 的 JSX 與 React 撰寫互動元件
  - 透過即時編輯器分享程式碼，讓使用者當場愛上您的產品
- 🔍 **搜尋功能**：全站內容皆可搜尋
- 💾 **文件版本控制**：保持文件與專案版本同步
- 🌍 **國際化 (i18n)**：支援多語言站點翻譯

Docusaurus v2+ 天生就致力於為所有使用者提供友善的無障礙體驗與閃電般的速度。

- ⚡️ **極速效能**。Docusaurus v2+ 遵循 [PRPL 模式](https://developers.google.com/web/fundamentals/performance/prpl-pattern/)，確保內容載入快如閃電。
- 🦖 **無障礙設計**。注重可訪問性，讓所有使用者都能平等使用您的網站。

## 設計原則 {#design-principles}

- **低學習門檻**。Docusaurus 應易於學習與使用，因其 API 相當精簡。即使需要更多程式碼與時間，大多數功能仍可實現。沒有抽象層比錯誤的抽象層更好，我們不希望使用者被迫繞過錯誤的抽象層。必看演講—[最小 API 表面積](https://www.youtube.com/watch?v=4anAwXYqLG8)。
- **直覺操作**。使用者在查看 Docusaurus 專案目錄或新增功能時不會感到不知所措。它應該看起來直觀且易於擴展，採用使用者熟悉的方法。
- **分層架構**。堆疊各層（內容/主題/樣式）之間的關注點分離應清晰——良好抽象且模組化。
- **合理預設值**。常見且受歡迎的效能優化與配置將為使用者預設完成，但仍可自行覆寫。
- **無廠商鎖定**。不強制使用預設插件或 CSS（但強烈建議）。核心基礎設施如 React Loadable 和 React Router 不可替換（因預設進行效能優化），但更高層級的組件可替換。Markdown 引擎、CSS 框架、CSS 方法論等架構選擇完全取決於使用者。

我們相信，作為開發者，了解函式庫的運作原理能幫助我們更熟練地使用它。因此我們投入心力解釋 Docusaurus 的架構與各組件，希望使用者閱讀後能更深入理解工具並提升使用熟練度。

## 與其他工具的比較 {#comparison-with-other-tools}

在所有靜態網站生成器中，Docusaurus 獨特專注於文件網站，並提供眾多開箱即用的功能。

我們也研究了其他主流靜態網站生成器，希望透過比較分析幫助您在眾多選擇中找到方向。

### Gatsby {#gatsby}

[Gatsby](https://www.gatsbyjs.com/) 具備豐富功能，擁有龐大的插件生態系統，能實現 Docusaurus 的所有功能。但這也伴隨著較高的學習曲線。Gatsby 在多方面表現出色，適合構建各類網站。而 Docusaurus 則專注於將一件事做到極致——成為編寫與發布內容的最佳工具。

GraphQL 是 Gatsby 的核心技術，但構建 Gatsby 網站並非必須使用 GraphQL。在大多數靜態網站開發場景中，您並不需要 GraphQL 提供的靈活性。

Docusaurus v2+ 的許多設計靈感來自 Gatsby 的優點，它是一個出色的替代方案。

[Docz](https://github.com/pedronauck/docz) 是基於 Gatsby 的主題，用於構建文檔網站。目前其功能尚不及 Docusaurus 完善。

### Next.js {#nextjs}

[Next.js](https://nextjs.org/) 是另一個熱門的混合式 React 框架。雖然它能協助構建優秀的文檔網站，但並未針對文檔使用場景進行專門設計，要實現 Docusaurus 開箱即用的功能需要投入更多開發工作。

[Nextra](https://github.com/shuding/nextra) 是基於 Next.js 的靜態網站生成器，具有特定設計理念。目前其功能尚不及 Docusaurus 全面。

### VuePress {#vuepress}

[VuePress](https://vuepress.vuejs.org/) 與 Docusaurus 有諸多相似之處——兩者都專注於內容導向的網站，並提供開箱即用的文檔功能。不同之處在於 VuePress 基於 Vue 構建，而 Docusaurus 基於 React。若您需要基於 Vue 的解決方案，VuePress 會是不錯的選擇。

### MkDocs {#mkdocs}

[MkDocs](https://www.mkdocs.org/) 是流行的 Python 靜態網站生成器，其核心價值與 Docusaurus 相似。

如果您不需要單頁應用程式且不計劃使用 React，這會是個好選擇。

[Material for MkDocs](https://squidfunk.github.io/mkdocs-material/) 是個精美的主題。

### Docsify {#docsify}

[Docsify](https://docsify.js.org/) 能輕鬆創建文檔網站，但它並非靜態網站生成器，也不利於 SEO 優化。

### GitBook {#gitbook}

[GitBook](https://www.gitbook.com/) 擁有簡潔設計，曾被眾多開源項目採用。隨著其發展重心轉向商業產品而非開源工具，許多功能已不再符合開源項目文檔站點的需求，因此不少用戶轉向其他方案。您可閱讀 [Redux 遷移至 Docusaurus 的討論](https://github.com/reduxjs/redux/issues/3161) 了解詳情。

目前 GitBook 僅對開源和非營利團隊免費，而 Docusaurus 對所有人免費開放。

### Jekyll {#jekyll}

[Jekyll](https://github.com/jekyll/jekyll) 是最成熟的靜態網站生成器之一——事實上在 Docusaurus 出現前，Facebook 多數開源網站都基於 Jekyll 構建！它的入門門檻極低。我們希望為開發者提供類似 Jekyll 的靜態網站構建體驗。

相較於通過 `<script />` 標籤添加交互性的靜態 HTML，Docusaurus 網站是 React 應用程式。借助現代 JavaScript 生態工具鏈，我們期望在文檔站點的效能、資源構建流程與優化、以及易用性方面樹立新標準。
