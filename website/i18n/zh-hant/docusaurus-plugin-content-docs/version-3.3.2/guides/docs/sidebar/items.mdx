# 側邊欄項目

```mdx-code-block
import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';
import BrowserWindow from '@site/src/components/BrowserWindow';
```

我們已在上一節的範例中介紹了三種項目類型：`doc`、`category` 和 `link`，其用法相當直觀。我們將正式介紹它們的 API。還有第四種類型：`autogenerated`，我們稍後會詳細說明。

- **[文件](#sidebar-item-doc)**：連結至文件頁面，並將其與側邊欄關聯
- **[連結](#sidebar-item-link)**：連結至任何內部或外部頁面
- **[分類](#sidebar-item-category)**：建立一個側邊欄項目的下拉選單
- **[自動生成](autogenerated.mdx)**：自動生成側邊欄片段
- **[HTML](#sidebar-item-html)**：在項目位置渲染純 HTML
- **[\*參照](multiple-sidebars.mdx#sidebar-item-ref)**：連結至文件頁面，但不讓該項目參與導航生成

## 文件：連結至文件 {#sidebar-item-doc}

使用 `doc` 類型來連結至文件頁面，並將該文件分配給側邊欄：

```ts
type SidebarItemDoc =
  // Normal syntax
  | {
      type: 'doc';
      id: string;
      label: string; // Sidebar label text
      className?: string; // Class name for sidebar label
      customProps?: Record<string, unknown>; // Custom props
    }

  // Shorthand syntax
  | string; // docId shortcut
```

範例：

```js title="sidebars.js"
export default {
  mySidebar: [
    // Normal syntax:
    // highlight-start
    {
      type: 'doc',
      id: 'doc1', // document ID
      label: 'Getting started', // sidebar label
    },
    // highlight-end

    // Shorthand syntax:
    // highlight-start
    'doc2', // document ID
    // highlight-end
  ],
};
```

如果您使用文件簡寫或[自動生成](autogenerated.mdx)的側邊欄，您將無法透過項目定義來自訂側邊欄標籤。然而，您可以在該文件中使用 `sidebar_label` Markdown 前置資料，其優先級高於側邊欄項目中的 `label` 鍵。同樣地，您可以使用 `sidebar_custom_props` 來宣告文件頁面的自訂元資料。

:::note

`doc` 項目設定了[隱含的側邊欄關聯](./multiple-sidebars.mdx#sidebar-association)。請勿將同一文件分配給多個側邊欄：請將類型更改為 `ref`。

:::

:::tip

側邊欄自訂屬性是將任意文件元資料傳播到客戶端的有用方式，因此在使用任何獲取文件物件的文件相關鉤子時，您可以獲得額外資訊。

:::

## 連結：連結至任何頁面 {#sidebar-item-link}

使用 `link` 類型來連結至任何非文件的頁面（內部或外部）。

```ts
type SidebarItemLink = {
  type: 'link';
  label: string;
  href: string;
  className?: string;
  description?: string;
};
```

範例：

```js title="sidebars.js"
export default {
  myLinksSidebar: [
    // highlight-start
    // External link
    {
      type: 'link',
      label: 'Facebook', // The link label
      href: 'https://facebook.com', // The external URL
    },
    // highlight-end

    // highlight-start
    // Internal link
    {
      type: 'link',
      label: 'Home', // The link label
      href: '/', // The internal path
    },
    // highlight-end
  ],
};
```

## HTML：渲染自訂標記 {#sidebar-item-html}

使用 `html` 類型在項目的 `<li>` 標籤內渲染自訂 HTML。

這對於插入自訂項目（如分隔線、區段標題、廣告和圖片）非常有用。

```ts
type SidebarItemHtml = {
  type: 'html';
  value: string;
  defaultStyle?: boolean; // Use default menu item styles
  className?: string;
};
```

範例：

```js title="sidebars.js"
export default {
  myHtmlSidebar: [
    // highlight-start
    {
      type: 'html',
      value: '<img src="sponsor.png" alt="Sponsor" />', // The HTML to be rendered
      defaultStyle: true, // Use the default menu item styling
    },
    // highlight-end
  ],
};
```

:::tip

選單項目已包裝在 `<li>` 標籤中，因此如果您的自訂項目很簡單（例如標題），只需提供一個字串作為值，並使用 `className` 屬性來設定樣式：

```js title="sidebars.js"
export default {
  myHtmlSidebar: [
    {
      type: 'html',
      value: '核心概念',
      className: 'sidebar-title',
    },
  ],
};
```

:::

## 分類：建立層次結構 {#sidebar-item-category}

使用 `category` 類型來建立側邊欄項目的層次結構。

```ts
type SidebarItemCategory = {
  type: 'category';
  label: string; // Sidebar label text.
  items: SidebarItem[]; // Array of sidebar items.
  className?: string;
  description?: string;

  // Category options:
  collapsible: boolean; // Set the category to be collapsible
  collapsed: boolean; // Set the category to be initially collapsed or open by default
  link: SidebarItemCategoryLinkDoc | SidebarItemCategoryLinkGeneratedIndex;
};
```

範例：

```js title="sidebars.js"
export default {
  docs: [
    {
      type: 'category',
      label: 'Guides',
      collapsible: true,
      collapsed: false,
      items: [
        'creating-pages',
        {
          type: 'category',
          label: 'Docs',
          items: ['introduction', 'sidebar', 'markdown-features', 'versioning'],
        },
      ],
    },
  ],
};
```

:::tip

當您不需要自訂時，請使用[**簡寫語法**](#category-shorthand)：

```js title="sidebars.js"
export default {
  docs: {
    指南: [
      'creating-pages',
      {
        文件: ['introduction', 'sidebar', 'markdown-features', 'versioning'],
      },
    ],
  },
};
```

:::

### 分類連結 {#category-link}

透過分類連結，點擊分類可導航至其他頁面。

:::tip

使用分類連結來引入一組文件。

自動生成的分類可利用 [`_category_.yml`](./autogenerated.mdx#category-item-metadata) 檔案宣告連結目標。

:::

#### 自動生成索引頁面 {#generated-index-page}

您可以自動生成一個索引頁面，顯示該分類下的所有直屬子項目。透過 `slug` 可自訂生成頁面的路由路徑，預設為 `/category/[categoryName]`。

```js title="sidebars.js"
export default {
  docs: [
    {
      type: 'category',
      label: 'Guides',
      // highlight-start
      link: {
        type: 'generated-index',
        title: 'Docusaurus Guides',
        description: 'Learn about the most important Docusaurus concepts!',
        slug: '/category/docusaurus-guides',
        keywords: ['guides'],
        image: '/img/docusaurus.png',
      },
      // highlight-end
      items: ['pages', 'docs', 'blog', 'search'],
    },
  ],
};
```

實際效果請參閱 [Docusaurus 指南頁面](/docs/category/guides)。

:::tip

使用 `generated-index` 連結作為快速建立入門文件的捷徑。

:::

#### 文件連結 {#category-doc-link}

分類可連結至現有文件。

```js title="sidebars.js"
export default {
  docs: [
    {
      type: 'category',
      label: 'Guides',
      // highlight-start
      link: {type: 'doc', id: 'introduction'},
      // highlight-end
      items: ['pages', 'docs', 'blog', 'search'],
    },
  ],
};
```

實際範例請見 [i18n 介紹頁面](../../../i18n/i18n-introduction.mdx)。

#### 在文件中嵌入自動生成索引 {#embedding-generated-index-in-doc-page}

您亦可透過 `DocCardList` 元件將自動生成的卡片列表嵌入一般文件頁面。該元件會顯示當前文件所屬父分類下的所有側邊欄項目。

```md title="docs/sidebar/index.md"
import DocCardList from '@theme/DocCardList';

<DocCardList />
```

```mdx-code-block
<BrowserWindow>

import DocCardList from '@theme/DocCardList';

<DocCardList />

</BrowserWindow>
```

### 可折疊分類 {#collapsible-categories}

我們支援分類的展開/折疊功能。預設情況下分類皆可折疊，但可透過設定 `collapsible: false` 禁用此功能。

```js title="sidebars.js"
export default {
  docs: [
    {
      type: 'category',
      label: 'Guides',
      items: [
        'creating-pages',
        {
          type: 'category',
          // highlight-next-line
          collapsible: false,
          label: 'Docs',
          items: ['introduction', 'sidebar', 'markdown-features', 'versioning'],
        },
      ],
    },
  ],
};
```

若要全域設定所有分類預設不可折疊，請在 `plugin-content-docs` 中將 `sidebarCollapsible` 選項設為 `false`：

```js title="docusaurus.config.js"
export default {
  presets: [
    [
      '@docusaurus/preset-classic',
      {
        docs: {
          // highlight-next-line
          sidebarCollapsible: false,
        },
      },
    ],
  ],
};
```

:::note

`sidebars.js` 中的設定會覆寫外掛配置，因此即使全域設定 `sidebarCollapsible` 為 `false`，仍可針對特定分類啟用折疊功能。

:::

### 預設展開分類 {#expanded-categories-by-default}

可折疊分類預設為收合狀態。若希望首次渲染時即展開，可設定 `collapsed` 為 `false`：

```js title="sidebars.js"
export default {
  docs: {
    Guides: [
      'creating-pages',
      {
        type: 'category',
        label: 'Docs',
        // highlight-next-line
        collapsed: false,
        items: ['markdown-features', 'sidebar', 'versioning'],
      },
    ],
  },
};
```

與 `collapsible` 類似，您也可將全域配置 `options.sidebarCollapsed` 設為 `false`。個別在 `sidebars.js` 設定的 `collapsed` 選項仍會優先於此配置。

```js title="docusaurus.config.js"
export default {
  presets: [
    [
      '@docusaurus/preset-classic',
      {
        docs: {
          // highlight-next-line
          sidebarCollapsed: false,
        },
      },
    ],
  ],
};
```

:::warning

當分類同時設定 `collapsed: true` 與 `collapsible: false`（無論透過 `sidebars.js` 或外掛配置），後者將優先生效，分類仍會以展開狀態渲染。

:::

## 使用簡寫語法 {#using-shorthands}

對於自訂需求較少的常見側邊欄項目，您可使用**簡寫語法**來精簡表達。此功能分為兩部分：[**文件簡寫**](#doc-shorthand) 與 [**分類簡寫**](#category-shorthand)。

### 文件簡寫 {#doc-shorthand}

類型為 `doc` 的項目可直接用字串形式表示其 ID：

```mdx-code-block
<Tabs>
<TabItem value="Longhand">
```

```js title="sidebars.js"
export default {
  sidebar: [
    // highlight-start
    {
      type: 'doc',
      id: 'myDoc',
    },
    // highlight-end
  ],
};
```

```mdx-code-block
</TabItem>
<TabItem value="Shorthand">
```

```js title="sidebars.js"
export default {
  sidebar: [
    // highlight-start
    'myDoc',
    // highlight-end
  ],
};
```

```mdx-code-block
</TabItem>
</Tabs>
```

因此前述範例可簡化為：

```js title="sidebars.js"
export default {
  mySidebar: [
    {
      type: 'category',
      label: 'Getting Started',
      items: [
        // highlight-next-line
        'doc1',
      ],
    },
    {
      type: 'category',
      label: 'Docusaurus',
      items: [
        // highlight-start
        'doc2',
        'doc3',
        // highlight-end
      ],
    },
    {
      type: 'link',
      label: 'Learn more',
      href: 'https://example.com',
    },
  ],
};
```

### 分類簡寫 {#category-shorthand}

一個分類項目可以用一個物件來表示，其鍵為標籤，值為子項目陣列。

```mdx-code-block
<Tabs>
<TabItem value="Longhand">
```

```js title="sidebars.js"
export default {
  sidebar: [
    // highlight-start
    {
      type: 'category',
      label: 'Getting started',
      items: ['doc1', 'doc2'],
    },
    // highlight-end
  ],
};
```

```mdx-code-block
</TabItem>
<TabItem value="Shorthand">
```

```js title="sidebars.js"
export default {
  sidebar: [
    // highlight-start
    {
      'Getting started': ['doc1', 'doc2'],
    },
    // highlight-end
  ],
};
```

```mdx-code-block
</TabItem>
</Tabs>
```

這讓我們可以將該範例簡化為：

```js title="sidebars.js"
export default {
  mySidebar: [
    // highlight-start
    {
      'Getting started': ['doc1'],
    },
    {
      Docusaurus: ['doc2', 'doc3'],
    },
    // highlight-end
    {
      type: 'link',
      label: 'Learn more',
      href: 'https://example.com',
    },
  ],
};
```

每個經過此轉換的簡寫物件將僅包含一個條目。現在考慮以下進一步簡化的範例：

```js title="sidebars.js"
export default {
  mySidebar: [
    // highlight-start
    {
      'Getting started': ['doc1'],
      Docusaurus: ['doc2', 'doc3'],
    },
    // highlight-end
    {
      type: 'link',
      label: 'Learn more',
      href: 'https://example.com',
    },
  ],
};
```

請注意兩個連續的分類簡寫如何被壓縮成一個包含兩個條目的物件。此語法生成一個**側邊欄片段**：你不應將該物件視為一個整體項目——此物件會被展開，每個條目成為一個獨立項目，並與其他項目（本例中的「了解更多」連結）拼接在一起，形成最終的側邊欄層級。側邊欄片段在討論[自動生成側邊欄](autogenerated.mdx)時也很重要。

只要有一個被簡化為單一分類簡寫的項目陣列，你也可以省略該封裝陣列。

```mdx-code-block
<Tabs>
<TabItem value="Before">
```

```js title="sidebars.js"
export default {
  sidebar: [
    {
      'Getting started': ['doc1'],
      Docusaurus: [
        {
          'Basic guides': ['doc2', 'doc3'],
          'Advanced guides': ['doc4', 'doc5'],
        },
      ],
    },
  ],
};
```

```mdx-code-block
</TabItem>
<TabItem value="After">
```

```js title="sidebars.js"
export default {
  sidebar: {
    'Getting started': ['doc1'],
    Docusaurus: {
      'Basic guides': ['doc2', 'doc3'],
      'Advanced guides': ['doc4', 'doc5'],
    },
  },
};
```

```mdx-code-block
</TabItem>
</Tabs>
```
