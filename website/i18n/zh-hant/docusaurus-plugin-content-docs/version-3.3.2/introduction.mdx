# 簡介

⚡️ Docusaurus 能協助您快速打造出**精美的文件網站**。

💸 自建技術堆疊成本高昂。請**專注於內容創作**，只需撰寫 Markdown 檔案即可。

💥 準備好進階功能了嗎？可使用**版本控制、國際化、搜尋和主題客製化**等進階功能。

💅 查看**[精選 Docusaurus 網站](/showcase?tags=favorite)**獲取靈感，並閱讀一些**[用戶見證](https://x.com/sebastienlorber/timelines/1392048416872706049)**。

🧐 Docusaurus 是**靜態網站生成器**。它建構出具有快速客戶端導航的**單頁應用程式**，充分發揮 **React** 的互動特性。除內建**文件功能**外，也能用於建立**任何類型的網站**（個人網站、產品頁、部落格、行銷登陸頁等）。

## 快速入門 ⏱️ {#fast-track}

透過實作，**5分鐘**快速理解 Docusaurus！

新建 Docusaurus 網站並遵循**極簡**嵌入式教學。

安裝 [Node.js](https://nodejs.org/en/download/) 後建立新網站：

```bash
npx create-docusaurus@latest my-website classic
```

啟動網站：

```bash
cd my-website
npx docusaurus start
```

開啟 [`http://localhost:3000`](http://localhost:3000) 並跟隨教學。

:::tip

立即透過 **[docusaurus.new](https://docusaurus.new)** 在瀏覽器中測試 Docusaurus！

或線上閱讀 **[5分鐘教學](https://tutorial.docusaurus.io)**。

:::

## 從 v1 遷移 {#migrating-from-v1}

Docusaurus v2+ 是基於現代化工具鏈的全面重寫版本。自 [v2 正式發布](https://docusaurus.io/blog/2022/08/01/announcing-docusaurus-2.0) 後，我們強烈建議**使用 Docusaurus v2+ 而非已棄用的 v1 版**。

已有 [眾多用戶](/showcase) 採用 Docusaurus v2+（[趨勢分析](https://www.npmtrends.com/docusaurus-vs-@docusaurus/core)）。

**適用 Docusaurus v2+ 的情境：**

- :white_check_mark: 需要現代化 Jamstack 文件網站
- :white_check_mark: 需要具客戶端路由的單頁應用程式 (SPA)
- :white_check_mark: 需完整運用 React 與 MDX 功能
- :white_check_mark: 無需支援 IE11 瀏覽器

**適用 [Docusaurus v1](https://v1.docusaurus.io/) 的情境：**

- :x: 不需要單頁應用程式 (SPA)
- :x: 需支援 IE11（注意：IE [已終止支援](https://docs.microsoft.com/en-us/lifecycle/products/internet-explorer-11)）

現有 v1 用戶可參閱我們的[遷移指南](./migration/index.mdx)升級至 v2+。

## 功能特色 {#features}

Docusaurus 的設計高度重視開發者與貢獻者的使用體驗。

- ⚛️ **以 React 與💚打造**：
  - 透過 React 進行擴展與客製化
  - 透過自訂 React 元件完全掌控網站的瀏覽體驗
- **可插拔架構**：
  - 使用基礎模板快速建構網站，再透過進階功能與插件強化
  - 將插件開源以與社群共享
- ✂️ **開發者體驗**：
  - 立即開始撰寫文件
  - 通用配置入口點，便於貢獻者維護
  - 熱重載與閃電般快速的增量建構
  - 基於路由的程式碼與資料分割
  - 輕鬆部署至 GitHub Pages、Netlify、Vercel 等服務

我們的共同目標——幫助使用者快速找到所需內容並更好理解您的產品。我們分享最佳實踐，助您打造優質文件站點。

- 🎯 **SEO 友善**：
  - 為所有可能路徑靜態生成 HTML 檔案
  - 頁面專屬 SEO 優化，讓使用者直接找到相關問題的官方文件
- 📝 **基於 MDX**：
  - 在 Markdown 中嵌入 JSX 與 React 撰寫互動元件
  - 透過即時編輯器分享程式碼，讓使用者當場愛上您的產品
- 🔍 **搜尋功能**：全站內容皆可搜尋
- 💾 **文件版本控制**：保持文件與專案版本同步
- 🌍 **國際化 (i18n)**：支援多語言站點翻譯

Docusaurus v2+ 天生具備極致效能，並貼心照顧所有使用者的可訪問性需求。

- ⚡️ **閃電速度**。遵循 [PRPL 模式](https://developers.google.com/web/fundamentals/performance/prpl-pattern/)，確保內容極速載入
- 🦖 **無障礙設計**。注重可訪問性，讓所有使用者平等獲取內容

## 設計原則 {#design-principles}

- **低學習曲線**。API 簡潔易學，即使需要更多程式碼與時間，使用者仍能實現大多數功能。寧缺毋濫的抽象設計，避免使用者被迫繞過錯誤抽象層。必看演講—[最小 API 表面積](https://www.youtube.com/watch?v=4anAwXYqLG8)
- **直覺化**。查看專案目錄或新增功能時不會感到困惑，採用熟悉的方式直觀擴展
- **分層架構**。各層級（內容/主題/樣式）關注點分離明確，抽象完善且模組化
- **合理預設**。為常見場景預設性能優化配置，同時保留覆寫選項
- **無廠商鎖定**。不強制使用預設插件或 CSS（但強烈建議）。核心基礎設施如 React Loadable 和 React Router 因預設性能優化不可替換，但更高層級的 Markdown 引擎、CSS 框架等架構完全由使用者自主選擇

我們相信，了解工具運作原理能幫助開發者更熟練使用。因此我們詳細解釋 Docusaurus 的架構與各組件，期望使用者能深入理解並提升使用效率。

## 與其他工具比較 {#comparison-with-other-tools}

在所有靜態網站生成器中，Docusaurus 獨特聚焦於文件站點，並提供大量開箱即用功能。

我們也研究其他主流靜態網站生成器，希望透過比較分析幫助您在眾多選擇中找到方向。

### Gatsby {#gatsby}

[Gatsby](https://www.gatsbyjs.com/) 具備豐富功能與龐大插件生態系，能實現 Docusaurus 的所有功能，但代價是較高的學習曲線。Gatsby 擅長構建多種類型網站，而 Docusaurus 則專注於將一件事做到極致——成為編寫與發布內容的最佳工具。

GraphQL 是 Gatsby 的核心技術，但構建靜態網站時通常不需要其提供的靈活性。

Docusaurus v2+ 的許多設計靈感源自 Gatsby 的優點，是極佳的替代方案。

[Docz](https://github.com/pedronauck/docz) 是基於 Gatsby 的文檔網站主題，目前功能不如 Docusaurus 完善。

### Next.js {#nextjs}

[Next.js](https://nextjs.org/) 是熱門的混合式 React 框架，雖能構建文檔網站，但缺乏對文檔場景的專項優化，需自行實現 Docusaurus 的開箱即用功能。

[Nextra](https://github.com/shuding/nextra) 是基於 Next.js 的靜態網站生成器，目前功能完整度低於 Docusaurus。

### VitePress {#vitepress}

[VitePress](https://vitepress.dev/) 與 Docusaurus 相似，均專注內容型網站並提供文檔專屬功能，但前者基於 Vue 而後者基於 React。若需 Vue 技術棧，VitePress 是理想選擇。

### MkDocs {#mkdocs}

[MkDocs](https://www.mkdocs.org/) 是 Python 靜態網站生成器，核心價值與 Docusaurus 相近。

若不需單頁應用且無意使用 React，此為合適選項。

[Material for MkDocs](https://squidfunk.github.io/mkdocs-material/) 提供精美的視覺主題。

### Docsify {#docsify}

[Docsify](https://docsify.js.org/) 易於創建文檔網站，但非靜態生成且不利 SEO。

### GitBook {#gitbook}

[GitBook](https://www.gitbook.com/) 擁有簡潔設計，曾廣泛用於開源項目。因其轉向商業化產品，許多開源項目已遷移至其他方案（如 [Redux 遷移案例](https://github.com/reduxjs/redux/issues/3161)）。

現 GitBook 僅對開源與非營利團隊免費，而 Docusaurus 完全免費。

### Jekyll {#jekyll}

[Jekyll](https://github.com/jekyll/jekyll) 是最成熟的靜態網站生成器之一（Facebook 開源網站曾廣泛採用）。Docusaurus 旨在提供類似的開發體驗，同時通過現代 JavaScript 工具鏈提升性能、構建流程與易用性標準。

相較於 Jekyll 的靜態 HTML 與腳本標籤交互，Docusaurus 網站是 React 應用，在性能優化與現代化工作流方面更具優勢。
