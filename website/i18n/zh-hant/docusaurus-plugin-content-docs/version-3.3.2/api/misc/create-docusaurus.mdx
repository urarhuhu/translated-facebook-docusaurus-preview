# 📦 create-docusaurus

一個腳手架工具，幫助您快速建立功能完整的 Docusaurus 應用程式。

## 使用方法 {#usage}

```bash
npx create-docusaurus@latest [name] [template] [rootDir]
```

`name` 參數將同時作為網站路徑和所建立應用程式 package.json 中的 `name` 欄位。可以是絕對路徑或相對於 `rootDir` 的路徑。

`template` 參數可為以下選項之一：

- `classic`: 使用經典模板（推薦）
- `facebook`: 使用 Facebook/Meta 模板，包含 Meta 特定的設定
- Git 倉庫 URL（以 `https://` 或 `git@` 開頭），可克隆至目標位置
- 相對於當前工作目錄的本地文件路徑，該路徑包含要複製到目標位置的文件

`rootDir` 將用於解析網站目錄的絕對路徑。預設為當前工作目錄。

:::warning

建議在互動式 shell 中使用此命令以確保所有功能可用。

:::

## 選項 {#options}

### `-t, --typescript` {#typescript}

當模板參數為可識別名稱時使用。目前僅 `classic` 模板提供 TypeScript 變體。

### `-g, --git-strategy` {#git-strategy}

當模板參數為 Git 倉庫時使用。必須為以下選項之一：

- `deep`: 保留完整 Git 歷史記錄
- `shallow`: 使用 `--depth=1` 進行克隆
- `copy`: 執行淺層克隆，但不建立 Git 倉庫
- `custom`: 輸入自訂 Git 克隆指令。我們將提示您輸入。您可以輸入類似 `git clone --depth 10` 的指令，我們會附加倉庫 URL 和目標目錄。

### `-p, --package-manager` {#package-manager}

值應為 `npm`、`yarn`、`pnpm` 或 `bun` 之一。若未明確提供，Docusaurus 將根據以下條件推斷：

- 當前工作目錄中已存在的鎖定文件（例如在現有專案中建立網站時）
- 用於呼叫 `create-docusaurus` 的指令（例如 `npm init`、`npx`、`yarn create`、`bunx` 等）
- 若所有啟發式方法均不可用，則透過互動式提示選擇

### `-s, --skip-install` {#skip-install}

若提供此選項，Docusaurus 在建立應用程式後不會自動安裝相依套件。`--package-manager` 選項僅在實際安裝相依套件時有用。
