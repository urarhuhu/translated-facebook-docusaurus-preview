# 下一步是什麼？

恭喜！您現在已經理解 Docusaurus 的大部分核心功能。您已經：

- [使用頁面插件](./creating-pages.mdx) 建立獨立的 React / Markdown 頁面
- [使用文檔插件](./docs/docs-introduction.mdx) 建立文檔頁面。這包括 [配置側邊欄](./docs/sidebar/index.mdx)，甚至 [版本控制](./docs/versioning.mdx)
- [使用部落格插件](../blog.mdx) 建立功能完整的部落格
- 嘗試了 [一系列 Markdown 功能](./markdown-features/markdown-features-intro.mdx)，這些功能對所有內容插件都很有用
- [使用樣式表](../styling-layout.mdx) 或 [swizzling](../swizzling.mdx) 來自訂網站外觀
- [在頁面中嵌入圖片和其他靜態資源](../static-assets.mdx)
- [為網站添加搜尋功能](../search.mdx)
- 了解如何透過標準 Docusaurus API 實現 [瀏覽器支援](../browser-support.mdx) 和 [SEO](../seo.mdx)
- 學習如何安裝和配置 [個別插件](../using-plugins.mdx)
- [部署](../deployment.mdx) 網站到內容託管服務
- [國際化](../i18n/i18n-tutorial.mdx) 網站以支援多種語言

此時，您可能已經有一個龐大的 `docusaurus.config.js` 檔案了😄 然而，您還沒寫多少程式碼！大多數功能都是透過呼叫封裝好的 Docusaurus API 實現的。在繼續探索的過程中，您可以選擇三條路徑：

- 學習更進階的 Docusaurus 概念。這將幫助您更深入理解這些 API 的功能。
- 閱讀 [所有現有 API](../docusaurus-core.mdx) 的文檔。許多功能在指南中尚未涵蓋！
- 學習如何 [開發插件](../api/plugin-methods/README.mdx) 來擴展網站功能。
