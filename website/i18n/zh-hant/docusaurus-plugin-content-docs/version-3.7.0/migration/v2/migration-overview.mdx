# 概述

本文檔將引導您將現有的 Docusaurus 1 網站遷移至 Docusaurus 2。

我們盡可能簡化此過程，並提供遷移 CLI 工具。

## 主要差異 {#main-differences}

Docusaurus 1 是純粹的文件網站生成器，使用 React 作為伺服器端模板引擎，但不在瀏覽器中載入 React。

Docusaurus 2 從頭重建，生成單頁應用程式，在瀏覽器中充分發揮 React 的威力。它提供了更高的自訂性，同時保留了 Docusaurus 1 的最佳特性——易於上手、版本化文件和多語言支援。

除此之外，Docusaurus 2 是一個**高效的靜態網站生成器**，可快速建立常見的內容驅動網站（例如文件、部落格、產品登陸頁和行銷頁面等）。

雖然我們的主要重點仍是協助您完善文件，但 Docusaurus 2 本質上是一個 React 應用程式，因此可用於建立任何類型的網站。**Docusaurus 現在不僅限於文件網站，可用於建立任何網站。**

## Docusaurus 1 結構 {#docusaurus-1-structure}

您的 Docusaurus 1 網站應具有以下結構：

```bash
├── docs
└── website
    ├── blog
    ├── core
    │   └── Footer.js
    ├── package.json
    ├── pages
    ├── sidebars.json
    ├── siteConfig.js
    └── static
```

## Docusaurus 2 結構 {#docusaurus-2-structure}

遷移後，您的 Docusaurus 2 網站可能如下所示：

```sh
├── docs
└── website
    ├── blog
    ├── src
    │   ├── components
    │   ├── css
    │   └── pages
    ├── static
    ├── package.json
    ├── sidebars.json
    ├── docusaurus.config.js
```

:::info

此遷移不會變更 `/docs` 資料夾的位置，但 Docusaurus v2 網站通常將 `/docs` 資料夾放在 `/website` 內

遷移至 v2 後，您可以自由將 `/docs` 資料夾放在任何位置。

:::

## 遷移流程 {#migration-process}

要獲得功能完整的 Docusaurus 2 網站，需遷移多項內容：

- 套件
- CLI 指令
- 網站配置
- Markdown 檔案
- 側邊欄檔案
- 頁面、元件和 CSS
- 版本化文件
- 多語言支援 🚧

## 自動化遷移流程 {#automated-migration-process}

[遷移 CLI](./migration-automated.mdx) 將為您處理遷移中的許多事項。

然而，部分內容難以自動化，您仍需手動完成。

:::note

建議先執行遷移 CLI，再透過手動遷移流程完成剩餘部分。

:::

## 手動遷移流程 {#manual-migration-process}

部分遷移內容（特別是頁面）無法自動化，需手動遷移。

[手動遷移指南](./migration-manual.mdx) 將提供所有手動步驟。
