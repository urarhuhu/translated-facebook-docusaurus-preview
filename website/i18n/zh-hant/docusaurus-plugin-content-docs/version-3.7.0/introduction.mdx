# 介紹

⚡️ Docusaurus 能協助您快速打造出**美觀的技術文件網站**。

💸 自建技術堆疊成本高昂。請專注於**內容創作**，只需撰寫 Markdown 檔案即可。

💥 需要更多功能？可使用**進階特性**如版本控制、國際化、搜尋與主題客製化。

💅 查看**[精選 Docusaurus 網站](/showcase?tags=favorite)**獲取靈感。

🧐 Docusaurus 是**靜態網站生成器**。它建構具備快速客戶端導航的**單頁應用程式**，充分發揮 **React** 的互動能力。除預設的**技術文件功能**外，也能用於建立**任何類型網站**（個人網站、產品頁、部落格、行銷登陸頁等）。

![](/img/slash-introducing.svg)

## 快速入門 ⏱️ {#fast-track}

透過實作，**5分鐘**快速理解 Docusaurus！

新建 Docusaurus 網站並遵循**極簡**內建教學。

安裝 [Node.js](https://nodejs.org/en/download/) 後建立新站點：

```bash
npx create-docusaurus@latest my-website classic
```

啟動站點：

```bash
cd my-website
npx docusaurus start
```

開啟 [`http://localhost:3000`](http://localhost:3000) 跟隨教學操作。

:::tip

立即透過 **[docusaurus.new](https://docusaurus.new)** 在瀏覽器中測試 Docusaurus！

或線上閱讀 **[5分鐘教學](https://tutorial.docusaurus.io)**。

:::

## 從 v1 遷移 {#migrating-from-v1}

Docusaurus v2+ 是基於現代化工具鏈的完全重寫版本。自 [v2 正式發布](https://docusaurus.io/blog/2022/08/01/announcing-docusaurus-2.0) 後，我們強烈建議**使用 v2+ 而非已停止維護的 v1 版本**。

已有 [眾多用戶](/showcase) 採用 Docusaurus v2+（[趨勢分析](https://www.npmtrends.com/docusaurus-vs-@docusaurus/core)）。

**適用 Docusaurus v2+ 的情境：**

- :white_check_mark: 需要現代化 Jamstack 技術文件網站
- :white_check_mark: 需要具備客戶端路由的單頁應用程式 (SPA)
- :white_check_mark: 需完整運用 React 與 MDX 功能
- :white_check_mark: 不需支援 IE11 瀏覽器

**適用 [Docusaurus v1](https://v1.docusaurus.io/) 的情境：**

- :x: 不需要單頁應用程式 (SPA)
- :x: 需支援 IE11（注意：IE [已停止支援](https://docs.microsoft.com/en-us/lifecycle/products/internet-explorer-11)）

現有 v1 用戶可參閱我們的[遷移指南](./migration/index.mdx)升級至 v2+。

## 功能特性 {#features}

Docusaurus 以高度重視開發者與貢獻者體驗為核心打造。

- ⚛️ **以 💚 和 React 建構**：
  - 透過 React 擴展與自訂功能
  - 提供自訂 React 元件，全面掌控網站瀏覽體驗
- 🔌 **可插拔式架構**：
  - 使用基礎模板快速建站，再進階運用功能與插件
  - 開源您的插件與社群共享
- ✂️ **開發者體驗**：
  - 立即開始撰寫文件
  - 統一配置入口提升協作者維護性
  - 變更時支援熱重載與閃電級增量建置
  - 基於路由的程式碼與資料拆分
  - 輕鬆部署至 GitHub Pages、Netlify、Vercel 等服務

我們的共同目標——協助使用者快速找到所需內容並深入理解您的產品。我們分享最佳實踐，助您打造高品質文件站點。

- 🎯 **SEO 友善**：
  - 為所有可能路徑靜態生成 HTML 檔案
  - 頁面專屬 SEO 優化，讓使用者直達解決問題的官方文件
- 📝 **基於 MDX 驅動**：
  - 在 Markdown 中嵌入 JSX 與 React 撰寫互動元件
  - 透過即時編輯器分享程式碼，讓使用者當場愛上您的產品
- 🔍 **全站搜尋**：完整內容皆可檢索
- 💾 **文件版本控制**：保持文件與專案版本同步
- 🌍 **國際化 (i18n)**：支援多語言站點翻譯

Docusaurus v2+ 天生具備極致效能，並以同理心為所有使用者提供無障礙體驗。

- ⚡️ **閃電般快速**。遵循 [PRPL 模式](https://developers.google.com/web/fundamentals/performance/prpl-pattern/)，確保內容極速載入
- 🦖 **無障礙設計**。注重可訪問性，讓所有使用者平等獲取內容

## 設計原則 {#design-principles}

- **低學習曲線**。Docusaurus API 簡潔易上手，即使需編寫更多程式碼，使用者仍能實現大多數功能。寧缺毋濫的抽象設計，避免使用者被迫繞過不當抽象層。必看演講—[最小化 API 表面積](https://www.youtube.com/watch?v=4anAwXYqLG8)
- **直覺化**。檢視專案目錄或新增功能時，使用者不會感到困惑。架構應符合直覺且易於擴展，採用熟悉的方式
- **分層架構**。各層級（內容/主題/樣式）關注點分離清晰——抽象完善且模組化
- **合理預設**。為常見場景預置效能優化與配置，同時保留覆寫選項
- **無廠商鎖定**。不強制使用預設插件或 CSS（但強烈建議）。核心基礎設施如 React Loadable 和 React Router 不可替換（因內建效能優化），但高階層可自由選擇。Markdown 引擎、CSS 框架、方法論等架構完全由使用者決定

我們相信，理解工具運作原理能讓開發者更精熟使用。因此我們致力解析 Docusaurus 架構與各元件，助使用者深入掌握工具精髓。

## 與其他工具比較 {#comparison-with-other-tools}

在所有靜態網站生成器中，Docusaurus 獨特聚焦於文件站點，提供大量開箱即用功能。

我們亦深入研究其他主流靜態網站生成器，分享比較見解，助您在眾多選擇中明智決策。

### Gatsby {#gatsby}

[Gatsby](https://www.gatsbyjs.com/) 具備豐富功能，擁有龐大的插件生態系統，能實現 Docusaurus 的所有功能。但這也意味著更高的學習曲線。Gatsby 在多方面表現出色，適合構建各種類型的網站。而 Docusaurus 則專注於將一件事做到極致——成為編寫和發布內容的最佳工具。

GraphQL 是 Gatsby 的核心技術之一，但構建 Gatsby 網站並非必須使用 GraphQL。在大多數靜態網站開發場景中，您並不需要 GraphQL 提供的靈活性。

Docusaurus v2+ 的許多設計靈感來自 Gatsby 的優秀特性，它是一個絕佳的替代方案。

[Docz](https://github.com/pedronauck/docz) 是基於 Gatsby 的主題，用於構建文檔網站。目前其功能尚不及 Docusaurus 完善。

### Next.js {#nextjs}

[Next.js](https://nextjs.org/) 是另一個熱門的混合式 React 框架。雖然它能幫助您構建優秀的文檔網站，但並未針對文檔使用場景進行專門設計，要實現 Docusaurus 開箱即用的功能需要更多開發工作。

[Nextra](https://github.com/shuding/nextra) 是基於 Next.js 的靜態網站生成器，具有特定設計理念。目前其功能尚不及 Docusaurus 完善。

### VitePress {#vitepress}

[VitePress](https://vitepress.dev/) 與 Docusaurus 有諸多相似之處——兩者都專注於內容型網站，並提供開箱即用的文檔功能。但 VitePress 基於 Vue 構建，而 Docusaurus 基於 React。如果您需要 Vue 技術棧的解決方案，VitePress 是不錯的選擇。

### MkDocs {#mkdocs}

[MkDocs](https://www.mkdocs.org/) 是流行的 Python 靜態網站生成器，其價值主張與 Docusaurus 相似。

如果您不需要單頁應用程式且不計劃使用 React，這是一個很好的選擇。

[Material for MkDocs](https://squidfunk.github.io/mkdocs-material/) 提供了精美的主題。

### Docsify {#docsify}

[Docsify](https://docsify.js.org/) 能輕鬆創建文檔網站，但它不是靜態網站生成器，也不利於 SEO 優化。

### GitBook {#gitbook}

[GitBook](https://www.gitbook.com/) 擁有簡潔的設計，曾被眾多開源項目採用。隨著其產品定位轉向商業化而非開源工具，許多功能已不再符合開源項目文檔站點的需求。因此許多項目已轉向其他方案。您可閱讀 [Redux 遷移至 Docusaurus 的討論](https://github.com/reduxjs/redux/issues/3161)了解詳情。

目前 GitBook 僅對開源和非營利團隊免費，而 Docusaurus 對所有人免費開放。

### Jekyll {#jekyll}

[Jekyll](https://github.com/jekyll/jekyll) 是最成熟的靜態網站生成器之一——事實上在 Docusaurus 之前，Facebook 多數開源網站都基於 Jekyll 構建！它的入門門檻極低。我們希望提供與 Jekyll 相似的開發體驗。

相較於使用靜態 HTML 和 `<script />` 標籤添加交互性的傳統方案，Docusaurus 網站是 React 應用程式。通過現代 JavaScript 生態工具鏈，我們期望在文檔站點性能、資源構建流程優化及易用性方面樹立新標準。
