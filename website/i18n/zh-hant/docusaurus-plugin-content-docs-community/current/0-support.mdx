# 支援

Docusaurus 擁有數千名開發者組成的社群。

本頁列出了您可以參與的 Docusaurus 相關社群；關於其他線上與實體的學習資源，請參閱本節的其他頁面。

在參與 Docusaurus 社群之前，[請先閱讀我們的行為準則](https://engineering.fb.com/codeofconduct/)。我們採用了《貢獻者公約》([Contributor Covenant](https://www.contributor-covenant.org/))，並期望所有社群成員遵守其中的準則。

## Stack Overflow {#stack-overflow}

Stack Overflow 是一個熱門的論壇，適合提出程式碼層級的問題或當您遇到特定錯誤時尋求協助。瀏覽標記為 **docusaurus** 的[現有問題](https://stackoverflow.com/questions/tagged/docusaurus)，或[提出您自己的問題](https://stackoverflow.com/questions/ask?tags=docusaurus)！

## 討論論壇 {#discussion-forums}

有許多線上論壇可供討論最佳實踐、應用程式架構以及 Docusaurus 的未來發展。如果您有可解答的程式碼層級問題，通常 Stack Overflow 會是更適合的選擇。

- [Docusaurus 線上聊天室](https://discord.gg/docusaurus)
  - [#help-and-questions](https://discord.gg/fwbcrQ3dHR) 提供使用者協助
  - [#contributors](https://discord.gg/6g6ASPA) 提供貢獻者協助
- [Reddit 上的 Docusaurus 社群](https://www.reddit.com/r/docusaurus/)

## 功能請求 {#feature-requests}

對於新功能請求，您可以在我們的[功能請求看板 (Canny)](/feature-requests)上發文，這是一個便於規劃路線的工具，並允許按投票數排序，這讓核心團隊能更清楚地了解哪些功能需求較高，相較於較難分類的 GitHub 議題。請避免直接為新功能（尤其是大型功能）提交 Pull Request，因為可能已經有人在開發或將成為我們的路線圖的一部分。請先與我們討論！

## 最新消息 {#news}

關於 Docusaurus 的最新消息，請在 X 上[追蹤 **@docusaurus**](https://x.com/docusaurus) 以及本網站的[官方 Docusaurus 部落格](/blog)。
