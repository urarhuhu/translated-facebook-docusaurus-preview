**2020年對Docusaurus而言是豐收的一年**，儘管[年初發生了令人遺憾的事件](https://docusaurus.io/blog/2020/01/07/tribute-to-endi)，且對所有人來說這都是動盪的一年。

我們持續投入[Docusaurus 2](https://docusaurus.io/)的開發，使其功能達到與[Docusaurus 1](https://v1.docusaurus.io/)的**完全對等**。

現在我們**推薦Docusaurus 2**作為新專案的**預設選擇**，並鼓勵v1用戶[遷移至Docusaurus 2](https://docusaurus.io/docs/migration)。

目前仍處於**alpha階段**，但預計**很快會有好消息公布**！

## Docusaurus 2年度亮點

今年我們開發了多項功能，以下是其中最重要的幾項：

- **國際化(i18n)**：輕鬆翻譯您的Docusaurus網站，即將發布！([預覽](https://github.com/facebook/docusaurus/pull/3325), [文件](https://docusaurus.io/docs/next/i18n/introduction))
- **版本下拉選單**：實現情境式版本切換
- **版本控制選項**：解決用戶在版本控制方面最常見的痛點
- **情境式搜索**：在當前版本中搜索，使用全新的DocSearch 3體驗
- **Markdown頁面**：使用Markdown而非React組件來創建獨立頁面
- **更優質的主題**：多項UI、UX和無障礙功能改進
- **漸進式網頁應用(PWA)**：為網站添加離線支持，並可安裝到設備
- **快速失敗機制**：嚴格的配置驗證、檢測失效鏈接，防止不良生產環境部署
- **多實例支持**：在單一網站上使用兩次文檔插件（例如iOS/Android SDK文檔）
- **遷移CLI工具**：自動化從Docusaurus 1升級的流程
- **CodeSandbox整合**：通過[new.docusaurus.io](https://new.docusaurus.io/)直接在瀏覽器中評估Docusaurus
- **Canary版本**：使用`@canary` npm標籤搶先體驗即將發布的版本
- **TypeScript支持**：內部代碼逐步採用TypeScript，並改善用戶使用體驗
- **開源Infima**：現已[開放原始碼](https://github.com/facebookincubator/infima)

## GitHub 動態

- **星標數**：14632 → 20946（年增 43.2%）
- **總貢獻者**：303 → 512（年增 68.9%），其中多數為非 Facebook 貢獻者
- **每週 npm 下載量**：2356 → 25592（年增 986%）
- **GitHub 上**，Docusaurus 1 被 6311 個專案使用（年增 62.9%），Docusaurus 2 則被 5039 個專案使用（年增 1940%）

## 與 Major League Hacking 的合作

我們已迎接 [Major League Hacking](https://mlh.io/)（MLH）學員參與了兩個賽季。

非常感謝他們帶來的**多項貢獻**，例如：

- 編寫 v1 至 v2 遷移 CLI 的基礎架構
- 協助將 React Native 網站從 v1 遷移至 v2
- 強化配置驗證的嚴格性與完整性，回傳更有幫助的錯誤訊息
- 新增 CodeSandbox 支援功能
- 改進 CI 流程：建置檔案大小與 Lighthouse 機器人

我們期待在 2021 年持續這項合作。

## 社群

Docusaurus 社群持續成長，[Discord](https://discord.gg/docusaurus) 伺服器相當活躍，且 [Stack Overflow 上的提問](https://stackoverflow.com/questions/tagged/docusaurus)也不斷增加。

Docusaurus 2 的**模組化架構**讓社群能夠開發並發布[第三方外掛](https://docusaurus.io/community/resources#community-plugins-)。為了更好地整合社群資源，如果您正在開發外掛，請[告知我們](https://github.com/facebook/docusaurus/discussions/4025)。

## 下一步是什麼？

隨著 **Docusaurus 2 的核心功能**終於完成，我們將能夠投入更多時間解決社群回報的痛點和錯誤，讓 Docusaurus 足夠穩定與便利，進入**測試版和候選發布階段**。

透過對**國際化與版本控制**的完善支援，以及**大規模遷移**如 **React Native** 和 **Jest**，我們現在為所有 Docusaurus 1 網站提供了清晰的遷移路徑。

我們仍有幾項**主要功能**希望繼續推進：

- [建立主題畫廊](https://github.com/facebook/docusaurus/issues/3522)，讓切換主題更輕鬆，並提供**一流的 Tailwind 支援**
- [升級至 Webpack 5](https://github.com/facebook/docusaurus/issues/4027)，提升效能與建置速度
- [加強與 CommonMark 的相容性](https://github.com/facebook/docusaurus/issues/3018)，因 MDX 使得基於 CommonMark 的網站較難採用 Docusaurus
- [升級至 MDX 2.0](https://github.com/facebook/docusaurus/issues/4029)
- 其他可能在 2021 年發現的任務

衷心感謝社群在 2020 年的[貢獻](https://github.com/facebook/docusaurus/graphs/contributors?from=2020-01-01&to=2021-01-01&type=c)，特別是：
